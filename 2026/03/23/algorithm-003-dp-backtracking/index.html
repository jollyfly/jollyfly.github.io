题目十一：最长回文子串（LeetCode 5）

题目描述

给你一个字符串 s，找到 s 中最长的回文子串。

示例：

分类标签

维度 标签
算法 中心扩展、动态规划
难度 中等
数据结构 字符串
关联 Manacher 算法（线性最优）

初步思路（暴力枚举）

思路： 枚举所有子串，判断是否回文，取最长。

问题： 双重循环 × 回文判断 = O(n³)，数据规模稍大即超时。

    end

优化过程

动态规划解法： 令 dp[i][j] 表示子串 s[i..j] 是否为回文。

状态转移：

  • dp[i][j] = (s[i] == s[j]) && (j - i < 3 || dp[i+1][j-1])
  • 边界：j - i < 3 时，两个字符或三个字符直接比较即可

填表顺序：从短到长，即按 j-i 递增遍历。

最优解：中心扩展法（O(n²) / O(1)）

核心思想：每个回文中心最多扩展 O(n) 次，总共 O(n) 个中心。

    flowchart TD
    A["遍历每个位置 i<br/>作为奇数回文中心"] --> B["expand(left=i, right=i)<br/>向两侧扩展"]
    B --> C{"s[left] == s[right] ?"}
    C -->|是| D["记录回文 [left, right]<br/>left--, right++"]
    D --> C
    C -->|否| E["停止，返回最大回文"]
    F["遍历每对相邻间隙<br/>i 与 i+1 作为偶数中心"] --> G["expand(left=i, right=i+1)"]
    G --> H{"s[left] == s[right] ?"}
    H -->|是| I["记录回文 [left, right]<br/>left--, right++"]
    I --> H
    H -->|否| J["停止，返回最大回文"]
    E --> K["比较奇数/偶数结果，取最大"]
    J --> K

Java 最优实现

复杂度分析

解法 时间复杂度 空间复杂度
暴力枚举 O(n³) O(1)
动态规划 O(n²) O(n²)
中心扩展（最优） O(n²) O(1)
Manacher O(n) O(n)

题目十二：括号生成（LeetCode 22）

题目描述

数字 n 代表生成括号的对数，请你生成所有有效括号的组合。

示例：

分类标签

维度 标签
算法 回溯、DFS
难度 中等
数据结构 树（递归树）
关联 全排列、组合生成

初步思路（暴力/递归枚举）

思路： 从空字符串出发，每一步尝试加左括号或右括号，递归生成所有可能，再过滤无效的。

问题： 组合数为 C(2n, n)，约 4^n，过滤操作大量无效计算。

画图演示：递归树

    flowchart TD
    ROOT((" "))
    ROOT --> L1["("]
    L1 --> LL2["((  ✓继续"]
    L1 --> LR2["()  左<右"]

    LL2 --> LLL3["(((  ✓"]
    LL2 --> LLR3["(()  ✓"]
    LLL3 --> LLLL4["(((("]
    LLL3 --> LLLR4["((()"]
    LLR3 --> LLRL4["(()("]
    LLR3 --> LLRR4["(())"]

    LR2 --> LRL3["()(  ✓"]
    LR2 --> LRR3["())( ✗ 剪枝"]

    LRL3 --> LRLL4["()(("]
    LRL3 --> LRLR4["()()"]
    LRLL4 --> LRLLL5["()((("]
    LRLL4 --> LRLLR5["()(()"]

    style LRR3 fill:#ffcccc
    style LLLL4 fill:#ffcccc
    style LRLLL5 fill:#ffcccc
    style LRLLR5 fill:#ffcccc

    classDef valid fill:#ccffcc
    class LLRR4,LRLR4,LLLR3,LLRL4 valid

优化过程：剪枝回溯

剪枝条件：

  1. 左括号数量 < n：可以继续加左括号
  2. 右括号数量 < 左括号数量：可以继续加右括号

剪枝后生成树节点数从 2^(2n) 减少到 Catalan 数 C(2n,n)/(n+1)。

最优解：剪枝回溯

    flowchart TD
    A["回溯 start"] --> B{"left < n ?"}
    B -->|是| C["加 '('<br/>递归 backtrack(left+1, right)"]
    B -->|否| D["跳过加左括号"]
    C --> E{"right < left ?"}
    E -->|是| F["加 ')'<br/>递归 backtrack(left, right+1)"]
    E -->|否| G["跳过加右括号"]
    F --> H{"cur.length() == 2*n ?"}
    H -->|是| I["加入 result"]
    H -->|否| B
    D --> H2{"结束?"}
    G --> H

Java 最优实现

复杂度分析

指标 分析
时间复杂度 O(4^n / √n)，即 Catalan 数 C(2n, n)/(n+1)，剪枝后是卡特兰数级别
空间复杂度 O(n)，递归栈深度为 2n

题目十三：子集（LeetCode 78）

题目描述

给你一个整数数组 nums，数组中的元素互不相同，返回该数组所有可能的子集（幂集）。

示例：

分类标签

维度 标签
算法 回溯、位运算
难度 中等
数据结构 数组、树（决策树）
关联 子集 II（LeetCode 90）、组合总和

初步思路（暴力枚举）

思路： 对每个元素，选择或不选择，形成 2^n 种组合。

问题： 代码可读性一般，且对于 nums 长度很大时（>30）1<<nums.length 会溢出。

画图演示：决策树

以 nums = [1, 2, 3] 为例：

    flowchart TD
    ROOT(("[]"))
    ROOT --> L1["[1]"]
    ROOT --> R1["[] (跳过1)"]

    L1 --> LL2["[1,2]"]
    L1 --> LR2["[1] (跳过2)"]

    R1 --> RL2["[2]"]
    R1 --> RR2["[] (跳过2)"]

    LL2 --> LLL3["[1,2,3]"]
    LL2 --> LLR3["[1,2] (跳过3)"]

    LR2 --> LRL3["[1,3]"]
    LR2 --> LRR3["[1] (跳过3)"]

    RL2 --> RLL3["[2,3]"]
    RL2 --> RLR3["[2] (跳过3)"]

    RR2 --> RRL3["[3]"]
    RR2 --> RRR3["[] (跳过3)"]

    classDef leaf fill:#ccffcc
    class LLL3,LLR3,LRL3,LRR3,RLL3,RLR3,RRL3,RRR3 leaf

优化过程：回溯（增量构建）

每次递归只向后选择，避免重复（如 [1,2] 和 [2,1] 只生成一次）。

    flowchart TD
    A["backtrack(start=0, path=[])"] --> B["i=0: path+1 → [1]<br/>递归 i=1"]
    B --> C["i=1: path+2 → [1,2]<br/>递归 i=2"]
    C --> D["i=2: path+3 → [1,2,3]<br/>递归 i=3"]
    D --> E["i=3: 退出，返回"]
    C --> F["i=2: path保持[1]<br/>→ [1,3]"]
    B --> G["i=1: path=[], 跳过1<br/>→ [2,3]"]
    G --> H["i=2: path=[2]<br/>→ [3]"]

Java 最优实现

复杂度分析

指标 分析
时间复杂度 O(n × 2^n)，每个子集生成平均 O(n)
空间复杂度 O(n)，递归栈深度

题目十四：单词拆分（LeetCode 139）

题目描述

给你一个字符串 s 和一个字符串列表 wordDict 作为字典，判断 s 是否可以被空格拆分成一个或多个在字典中出现的单词。

示例：

分类标签

维度 标签
算法 动态规划、Trie（字典树）优化
难度 中等
数据结构 HashSet、Trie
关联 单词拆分 II、破碎路径

初步思路（暴力递归 + 记忆化）

思路： 从每个位置尝试切分，判断剩余子串是否可拆分。

问题： 每次循环都要遍历 wordDict，且 substring 创建大量中间字符串。优化方向：用 Trie 快速匹配前缀。

画图演示：DP 状态转移

    flowchart TD
    A["s = 'leetcode'<br/>wordDict = {leet, code}"]
    B["dp[0] = true（空串可拆分）"]
    B --> C["i=4: s[0..4)='leet'<br/>dp[0] && contains('leet') → dp[4]=true"]
    C --> D["i=8: s[0..8)='leetcode'<br/>dp[4] && contains('code') → dp[8]=true ✓"]
    E["dp数组状态转移表"]
    F["index: 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8"]
    G["dp值:  T | F| F| F| T | F| F| F| T "]
    E --> F
    E --> G
    F --- H["dp[4]=true 关键节点"]
    G --- I["dp[8]=true 最终答案"]

优化过程：Trie + DP

    flowchart LR
    subgraph Trie结构
        ROOT(("ROOT"))
        ROOT --> L["l"]
        L --> LE["e"]
        LE --> EET["e"]
        EET --> T["t *"]
        LE --> E["e"]
        E --> CO["c"]
        CO --> OD["o"]
        OD --> DE["de *"]
    end

    subgraph DP匹配流程
        S["s='leetcode'"] --> T1["Trie匹配前缀<br/>找到 'leet' 在 idx=4"]
        T1 --> S2["dp[4]=true<br/>从4继续匹配"]
        S2 --> T2["Trie匹配 'code'<br/>在 idx=8"]
        T2 --> RES["dp[8]=true ✓"]
    end

用 Trie 替代 wordDict.contains(prefix)，避免每次 O(n) 的 HashSet 遍历，改用 O(m) 的字符匹配（m 为词长）。

题目十五：排序颜色（LeetCode 75）

题目描述

给定一个包含 0、1、2 三种颜色的数组 nums，对它们进行排序，使相同颜色的元素相邻，并按照 0 → 1 → 2 的顺序排列。要求原地排序，且不能使用库函数。

示例：

分类标签

维度 标签
算法 荷兰国旗问题、三路快排 Partition
难度 中等
数据结构 数组
关联 快速排序、分桶排序

初步思路（计数排序）

思路： 先统计 0、1、2 的个数，再覆盖原数组。

问题： 需要两次遍历（统计 + 回填），且不满足”One-pass”的高标准。

画图演示：荷兰国旗三路分割

荷兰国旗问题（Dutch National Flag Problem）：将数组分为三段：

  • [0, zero]：全是 0
  • (zero, two)：全是 1（未处理区域）
  • [two, n-1]：全是 2
    flowchart LR
    subgraph 初始状态
        A0["nums: 2 0 2 1 1 0"]
        B0["idx: 0 1 2 3 4 5"]
    end

    subgraph 执行过程
        direction TB
        S1["初始: zero=0, i=0, two=n"]
        S1 --> S2{"nums[i] ?"}
        S2 -->|0| A3["swap(nums,i, zero)<br/>zero++, i++"]
        S2 -->|1| A4["i++ (不变)"]
        S2 -->|2| A5["swap(nums,i, two-1)<br/>two--, i不变"]
        A3 --> S2
        A4 --> S2
        A5 --> S2
        S2 -->|i == two| END["排序完成"]
    end

    subgraph 状态变化表
        T1["步骤 | i | zero | two | 数组状态"]
        T2["初始化 | 0 | 0 | 6 | 2 0 2 1 1 0"]
        T3["i=0, num=2 | 0 | 0 | 5 | 0 0 2 1 1 2"]
        T4["i=0, num=0 | 1 | 1 | 5 | 0 0 2 1 1 2"]
        T5["i=1, num=0 | 2 | 1 | 5 | 0 0 2 1 1 2"]
        T6["i=2, num=2 | 2 | 1 | 4 | 0 0 1 1 2 2"]
        T7["i=2, num=1 | 3 | 1 | 4 | 0 0 1 1 2 2"]
        T8["i=3, num=1 | 4 | 1 | 4 | 0 0 1 1 2 2"]
        T9["i=4, num=2 | 4 | 1 | 3 | 0 0 1 1 2 2"]
        T10["i==two 结束 | - | - | - | 0 0 1 1 2 2 ✓"]
        T1 --> T2 --> T3 --> T4 --> T5 --> T6 --> T7 --> T8 --> T9 --> T10
    end

最优解：三路快排 Partition（单次遍历）

    flowchart TD
    A["设置 three = n<br/>设置 zero = 0<br/>设置 i = 0"] --> B{"i < two ?"}
    B -->|是| C{"nums[i] 的值"}
    C -->|0| D["swap(nums, i, zero)<br/>zero++<br/>i++"]
    C -->|1| E["i++"]
    C -->|2| F["two--<br/>swap(nums, i, two)<br/>i 不变（需重新判断新 nums[i]）"]
    D --> B
    E --> B
    F --> B
    B -->|否| G["排序完成<br/>返回 nums"]

    classDef red fill:#ffaaaa

Java 最优实现

复杂度分析

方案 时间复杂度 空间复杂度 遍历次数
计数排序 O(n) O(1) 2 次
三路 Partition（最优） O(n) O(1) 1 次

关键细节：为什么 nums[i] == 2 时 i 不变？

因为从右侧交换过来的元素是未处理区域的元素（two 原来的位置），它可能是 0、1 或 2，需要重新判断。而 nums[i] == 0 时，从左侧交换过来的是已知为 1 的元素（因为 [zero, i) 区域都是 1），所以可以直接 i++。

总结对比

题号 题目 核心算法 时间 空间
11 最长回文子串 中心扩展 O(n²) O(1)
12 括号生成 剪枝回溯 O(4^n/√n) O(n)

共同套路：

  • 回溯：选/不选、增删路径、递归回溯三大步
  • DP：状态定义 → 转移方程 → 填表顺序
  • Partition：双指针/三路分割，原地 O(1) 空间是核心目标
