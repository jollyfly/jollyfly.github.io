项目概览

grocy 是一款开源、自托管的家庭物资与日常管理工具，被称为”家庭版 ERP”。它最初由德国开发者 Bernd Best 创建，旨在帮助家庭用户管理食品杂货、家庭用品、药品等物资的库存、过期时间和采购计划。

截至目前，grocy 在 GitHub 上已收获 22k+ Stars，是家庭自动化与个人管理领域最受欢迎的开源项目之一。

功能模块

grocy 围绕家庭日常管理场景，拆解出四个核心功能模块：

1. 购物清单（Shopping List）

用户可以从库存中快速添加需要补货的物品，或手动创建临时采购项。系统支持：

  • 自动补货建议：根据历史消耗速度预测下次购买时间
  • 清单分组：按超市分区（生鲜/日用品/冷冻等）对物品分组
  • 快速添加：支持条形码扫描和快捷键批量录入

2. 库存管理（Stock）

库存模块是 grocy 的核心，系统记录每个物品的：

  • 当前数量与最小库存阈值：低于阈值自动触发提醒
  • 保质期追踪：临近过期/已过期的物品高亮显示
  • 批量操作：支持批量入库、批量消费、批量调整数量
  • 使用位置管理：记录物品存放位置（冰箱/储物柜/车库等）

3. 食谱（Recipes）

食谱模块将食材管理与烹饪流程结合：

  • 食谱库：存储烹饪配方，支持配料表自动换算（按用餐人数）
  • 智能采购：选定食谱后自动计算所需食材，生成购物清单
  • 食材消耗联动：烹饪完成后自动从库存扣减相应数量

4. 设备维护（Equipment / Chores）

管理家庭设备和周期性任务：

  • 设备清单：记录家中电器的购买日期、保修期、维护周期
  • 周期任务：扫地机滤芯更换、空调滤网清洗等周期性维护自动生成待办
  • 日历视图：以月历方式展示所有待维护设备和任务

架构解析

grocy 的技术选型非常务实，采用经典的 PHP 后端 + Vue.js 前端 组合，配合 Docker 实现一键部署。

后端：PHP

grocy 的后端采用纯 PHP 开发，不依赖 Composer 生态中的重型框架，而是使用极简的 自定义 MVC 框架，核心代码量小、易审计。数据层支持三种数据库：

  • SQLite：默认方案，零配置，单文件存储，适合个人/家庭使用
  • PostgreSQL / MySQL：适合多用户或需要远程访问的场景

前端：Vue.js

前端使用 Vue.js 构建单页应用（SPA），通过 REST API 与后端交互。界面设计追求效率优先：

  • 键盘快捷键覆盖所有高频操作
  • 侧边栏导航 + 内容区布局，适合桌面端使用
  • 响应式设计，移动端也可访问

技术亮点

grocy 能在同类开源项目中脱颖而出，离不开以下技术决策：

1. 零外部依赖的后端架构

大多数 PHP 项目会引入 Symfony、Laravel 等框架，grocy 选择手撸轻量 MVC，核心依赖仅 PHP 原生扩展（pdo、json、date 等）。这带来的好处是：

  • 镜像体积小：Alpine 基础镜像 + PHP，约 80MB
  • 安全面小：攻击面极小，代码可完全掌握在自己手中
  • 启动极快：无需 Composer autoload 初始化

2. 离线优先的数据设计

grocy 默认使用 SQLite，所有数据存储在本地文件系统，而非云端。这意味着：

  • 完全离线可用，不依赖外部网络
  • 数据主权完全归用户，无隐私泄露风险
  • 多设备同步需依赖 OwnTracks、Syncthing 等工具自建

3. 以效率为导向的 UX 设计

grocy 的交互设计深受程序员青睐：

  • 键盘导航：几乎所有操作都有快捷键（+ 添加、e 编辑、d 删除、Enter 确认）
  • 批量操作：支持多选批量处理，减少重复点击
  • 快速搜索：全局搜索栏（/ 唤起）支持模糊匹配物品名称

4. 活跃的社区与丰富的插件生态

grocy 的插件机制允许开发者扩展功能：

  • Barcode Scanner：通过摄像头扫描条形码快速添加物品
  • grocy-desktop：Electron 封装，提供桌面通知和系统托盘
  • Home Assistant 集成：与智能家居平台联动

关键设计决策：

  • DI\Container（PHP-DI）负责所有服务的依赖注入，控制器通过构造器注入获取 Service 层实例
  • 视图层使用 Blade 模板引擎（SlimBladeView），支持模板缓存到 data/viewcache/

源码实现深度解析

核心文件结构

库存流水账设计（事件溯源简化版）

grocy 的”撤销”能力源于 stock_log 流水账表设计：

字段 含义
transaction_id 一次操作（如”购买 5 瓶酱油”）的全局 ID
transaction_type purchase / consume / transfer_from 等
product_id 产品 ID
amount 变动数量
stock_id 具体哪个库存批次（支持按件标记）

撤销时，根据 transaction_id 找到所有相关 stock_log 记录，执行反向操作即可恢复库存状态。这种设计比直接修改 stock 表更可靠，因为它保留了完整的操作历史。

总结

grocy 用极简的技术栈做了一件非常具体的事：让家庭物资管理变得有条理、可量化、自动化。它不追求大而全，而是把”库存管理”这一个点做到极致——到期提醒、自动补货、食谱联动，这些功能组合起来真正能减少”冰箱里那瓶酱什么时候开的”这类日常烦恼。

对于动手能力强、注重隐私的用户，grocy 是树莓派或 NAS 上的绝佳选择；对于开发者，它的代码结构简单易懂，是学习 PHP + Vue.js 完整项目开发的优秀参考。
