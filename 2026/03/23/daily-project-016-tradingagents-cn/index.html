项目概览

TradingAgents-CN（hsliuping/TradingAgents-CN）是 TradingAgents 项目的中文增强分支，基于多智能体 LLM 架构，专门针对中文金融市场进行深度优化。该项目利用大语言模型的推理与规划能力，构建了一套从市场分析、信号生成到交易执行的全链路自动化交易系统。

核心特性：

  • 🌐 原生中文支持：数据源、Prompt、报告输出全面中文适配
  • 🤖 多智能体协作：Researcher、Analyst、Trader 多角色分工明确
  • 📊 多市场覆盖：A 股、港股、美股数据源均有对应接入方案
  • 🔧 模块化设计：各组件解耦，便于二次开发和策略替换
  • 🛡️ 风控层内置：内置止损、仓位管理、风险预算模块

关键差异：原版面向北美散户/量化研究者，TradingAgents-CN 重构了数据层和 Prompt 层，更贴合国内量化开发者的实际工作环境。

多智能体架构（图解）

各智能体职责

  • Orchestrator：接收用户目标，协调多智能体工作流，决定调用顺序和并行策略。
  • Researcher：负责数据采集（行情、财经新闻、公司公告、宏观指标），做初步清洗和结构化。
  • Analyst：对结构化数据做深度分析，包括技术指标计算、基本面估值、资金流向、情绪分析等。
  • Trader：结合 Analyst 的输出生成交易信号，执行策略打分，最终输出买卖指令和仓位建议。
  • Risk Controller：独立于其他智能体运行，对每笔信号进行风控审核，必要时否决或修改指令。

核心原理分析

1. 多智能体协作框架（Multi-Agent Collaboration）

TradingAgents-CN 的核心架构基于任务分解 + 智能体协作的范式。当用户输入一个交易目标（如”分析茅台当前是否值得买入”），整个系统通过以下流程运转：

关键设计点：

  • Orchestrator（调度器）：不执行具体任务，只负责任务分发和流程控制。它根据任务类型决定并行还是串行调用子智能体，并汇总各智能体的输出形成最终决策。
  • 状态共享：各智能体通过共享的 context 字典交换中间结果，而非直接相互调用，降低耦合度。
  • 异步执行：Researcher 的数据获取耗时最长（网络 IO），Analyst 可以在 Researcher 仍在工作时就开始处理已到达的部分数据，实现流水线并行。

2. LLM 驱动的信号生成

传统量化系统依赖人工设定的因子模型（如 MACD、RSI、布林带），TradingAgents-CN 引入 LLM 作为因子合成器和决策引擎：

LLM 的优势在于能够综合考虑”行情数据 + 宏观叙事 + 情绪信号”，这是传统规则引擎难以量化的维度。但这也引入了幻觉风险——LLM 可能生成看似合理但实则错误的技术指标解读。

3. 中文金融语料适配

TradingAgents-CN 的中文增强并非简单翻译 Prompt，而是针对中文金融语料做了以下深度适配：

  • 财报格式差异处理：A股使用中国会计准则（CAS），与美股GAAP在科目名称、披露节奏上差异显著。系统内置了CAS→标准科目的映射表。
  • 中文情绪分析：利用中文 BERT/ERNIE 模型对东方财富股吧、同花顺资讯进行情感打分，生成中文舆情因子。
  • 政策事件注入：国内股市受政策影响极大，系统内置了”政策事件”维度的输入通道，LLM 在决策时会被引导考虑近期监管动态。

总结

TradingAgents-CN 将多智能体 LLM 架构带入中文量化交易场景，填补了原版 TradingAgents 在 A 股/港股数据生态上的空白。其优势在于：

  • ✅ 开箱即用：数据采集→分析→信号全链路打通
  • ✅ 中文优先：Prompt 和输出全程中文，降低使用门槛
  • ✅ 风控内置：独立风控层，避免 LLM 幻觉导致的失控交易
  • ✅ 模块解耦：各层可独立替换，适应不同机构的技术栈

适用场景：个人量化研究者、量化私募的 AI 策略原型、LLM+金融的学术研究。

需要注意：LLM 生成交易信号仍存在幻觉风险，强烈建议在回测充分验证后再接入实盘，且始终开启风控层。
