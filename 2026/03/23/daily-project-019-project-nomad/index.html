定位 离线生存计算机

核心理念：一台自包含的离线生存计算机，集成关键工具、知识库和 AI，无网络也能保持运转——随时随地为你提供信息支持和决策辅助。

离线生存计算机概念

传统应用依赖云端，一旦没有网络就成了废铁。N.O.M.A.D 反其道而行：把一切你需要的信息和工具，全部打包进一个可离线运行的 Docker 容器里。

核心功能模块

1. 离线知识库

预置荒野求生、医疗急救、食物辨别、水源净化等百科内容，不依赖维基百科或搜索引擎。

2. 本地 AI 推理

Docker 内置轻量级本地模型（Llama.cpp 或等效方案），可以在没有互联网的情况下进行自然语言问答。

3. 离线地图与导航

集成 OpenStreetMap 离线地图包，配合 GPS 定位，在没有蜂窝网络时依然知道自己的位置。

4. 应急工具集

  • 医疗急救：常见伤病的应急处理步骤
  • 野生动物识别：危险动物辨别
  • 食物/植物辨别：可食用 vs 有毒
  • 水源净化指引：如何把野外水变成饮用水

5. 天气与天文

读取离线气象数据，提供野外环境预警；结合星图工具用于夜间导航。

知识库设计

知识库采用结构化 Markdown 文件存储，便于维护和扩展：

AI 在离线环境下直接读取本地 Markdown 文件进行 RAG 检索，不依赖任何外部 API。

AI 在离线场景的应用

场景 传统方案 N.O.M.A.D 方案
受伤急救 翻纸质手册（慢） 问 AI，快速定位步骤
未知植物 凭经验/猜 拍照 + 本地视觉模型识别
迷失方向 纸质地图（需事先打印） GPS + 离线地图 + AI 路线建议
天气判断 看云的经验 气压计数据 + AI 气象分析

技术架构

  • 前端：Web 界面，可在任何设备浏览器打开
  • 后端：Express.js，REST API
  • AI：本地 llm（llama.cpp 或等效），无外网调用
  • 知识库：本地 Markdown + 简单向量检索（或 BM25）
  • 地图：OpenStreetMap 离线包

部署方式

离线使用：提前在有网络时构建 Docker 镜像，之后断网运行。

适用人群

  • 野外作业人员：地质勘探、林业、救援队
  • 户外爱好者：徒步、登山、航海
  • 极简主义者：希望关键工具不依赖网络
  • 应急准备者：家庭应急包中的数字备份

核心原理分析

1. 离线优先架构（Offline-First Architecture）

N.O.M.A.D 的核心理念是**”网络是奢侈品，信息是刚需”**。整个系统的架构设计始终围绕”无网络也能运行”这一约束展开：

离线实现的关键：

  • 预下载 Docker 镜像：安装时一次性拉取所有镜像，安装完成后断网也能运行
  • 本地模型推理：使用 Ollama + llama.cpp 在本地运行 LLM，零网络依赖
  • 离线知识库：Kiwix 格式打包的 Wikipedia 内容（.zim 文件），单文件可达数十 GB

总结

Project N.O.M.A.D 是一个非常有社会价值的开源项目。它解决的不是技术难题，而是真实场景中的信息鸿沟——在网络最不可靠的地方（野外、灾区），恰恰最需要可靠的信息支持。

架构上采用 Docker 容器化 + 本地 AI 推理 + 离线知识库，是一套工程化程度很高的离线优先设计方案。对于研究”如何在无网络环境下运行 AI 应用”的开发者来说，这个项目提供了很好的参考范本。
