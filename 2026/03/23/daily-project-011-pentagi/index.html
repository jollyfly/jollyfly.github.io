背景与动机

传统的渗透测试高度依赖人工经验，从信息收集、漏洞扫描到权限提升，整个流程不仅耗时，而且对测试人员的知识广度有极高要求。随着大语言模型（LLM）的崛起，将 AI Agent 引入渗透测试全流程成为安全社区探索的前沿方向。

Pentagi 正是这一方向的实践者——它构建了一套多智能体协作架构，让多个 AI Agent 各司其职、自主协同，完成从侦察到报告的全链路渗透测试任务。

核心架构

工作流程

  1. 任务规划（Task Planner）：接收测试目标，LLM 拆解为可执行子任务
  2. 多 Agent 并行执行：Recon / Vuln / Exploit Agent 各自调用专用工具链
  3. 工具注册表（Tool Registry）：统一管理扫描器、字典、PoC 等工具的调用接口
  4. 执行器（Executor）：负责任务调度、状态追踪与错误重试
  5. 报告生成（Reporter）：汇总发现，自动输出结构化渗透测试报告

核心功能解析

1. 全自动侦察（Recon）

Pentagi 内置多维度侦察引擎：

  • 被动信息收集：利用 OSINT 手段获取域名、邮箱、关联资产
  • 主动端口扫描：集成 Nmap Masscan 等工具，自动识别开放服务
  • Web 指纹识别：自动探测 Web 技术栈、CMS、中间件版本

2. AI 驱动的漏洞分析

传统扫描器依赖规则库，而 Pentagi 的 Vuln Agent 利用 LLM 的推理能力：

  • 结合上下文判断漏洞真实可利用性（减少误报）
  • 针对目标资产特点生成定制化探测策略
  • 支持自定义漏洞检测逻辑的动态注入

3. 智能漏洞利用（Exploit）

Exploit Agent 并非简单调用公开 PoC，而是：

  • 基于漏洞分析结果，LLM 推理最优利用路径
  • 自动适配目标环境（版本、配置、WAF 规避）
  • 支持自定义利用脚本的即插即用

4. 实时监控与日志

  • 全程任务状态可视化
  • 详细操作日志可审计、可回放
  • 支持中断恢复，任务不因会话断开而丢失

5. 报告自动生成

测试完成后，Reporter Agent 自动整合：

  • 发现的所有漏洞（含 CVSS 评分）
  • 利用路径图（attack path visualization）
  • 修复建议（按优先级排序）

核心原理分析

1. Docker 沙箱隔离架构

Pentagi 最核心的设计哲学是安全隔离——所有渗透测试操作都在 Docker 容器内执行，避免对宿主机造成不可逆的损害。

隔离网络原理：Worker 容器通过 Docker bridge 网络与宿主机隔离，但可通过 network_mode: host 或专用网卡访问测试目标网络。这种”隔离但可配置网络边界”的设计，让渗透测试流量可控，不会误伤内网其他主机。

2. 多 Agent 协作与任务分片

Pentagi 的 Agent 协作采用 MAP（Map-And-Reduce） 模式：

每个子 Agent 都是一个独立的 LLM 对话会话，拥有自己的 system prompt 和工具调用上下文。Agent 之间通过 PostgreSQL 数据库共享中间结果，而非直接内存通信——这使得任务可以在任意时刻中断、恢复，不会因进程崩溃而丢失。

3. Graphiti 知识图谱长期记忆

Pentagi 引入了 Graphiti（基于 Neo4j）构建渗透测试知识图谱：

在多轮渗透测试中，当 Agent 面对一个已识别过的目标时，知识图谱会优先提供历史发现，使测试过程呈现增量式而非每次从零开始。

4. 智能任务规划（Task Planner）

Task Planner 本质是一个垂直领域的 LLM Agent，专门针对渗透测试场景做了 fine-tuned 的 prompt 工程：

低 temperature（0.1）确保规划结果稳定可复现，高 risk_level 任务会被标记为由人工确认后再执行。

5. Langfuse 可观测性集成

Pentagi 通过 Langfuse 实现对整个 LLM 调用链路的追踪：

每个 LLM 调用（Recon 推理、Vuln 分析、Exploit 决策）都会生成一个 Langfuse trace，包含：

  • 输入/输出 token 数（成本追踪）
  • Latency（性能瓶颈定位）
  • Tool calls（工具调用链路的完整记录）
  • Scores（人工或自动的质量评分）

源码实现深度解析

1. Agent 基类设计

模板方法模式：所有 Agent 共享 think() 方法和 memory 管理逻辑，差异化行为由 run() 和 system_prompt 子类实现。这确保了 Agent 架构的一致性，同时支持高度定制。

2. 工具注册与动态调用

工具注册表的优势：新增安全工具只需一个装饰器，无需修改 Agent 代码。Agent 的 system_prompt 会自动从注册表获取工具列表，实现工具发现自动化。

3. 任务队列与状态机

状态机设计：每个任务有明确的有限状态（PENDING → RUNNING → COMPLETED/FAILED/BLOCKED），配合 asyncio.Queue 实现高效的并发任务调度。BLOCKED → PENDING 的转换由依赖任务完成事件触发，无需轮询。

4. Docker Worker 远程执行

安全设计：

  • read_only=True + security_opt 防止容器内提权
  • mem_limit 和 cpu_quota 防止恶意工具耗尽系统资源
  • detach=True + wait(timeout=300) 确保工具执行不会无限挂起
  • remove(force=True) 在 finally 块确保容器清理，防止磁盘空间泄漏

技术亮点

多 Agent 协作机制

不同于单 Agent 系统，Pentagi 采用任务分片 + 异步协作模式，各 Agent 之间通过消息队列通信，避免单点阻塞。

总结

Pentagi 代表了 AI + 安全 融合的一个有意义的探索方向。它并非要取代安全工程师，而是将大量重复性、规则性的工作自动化，让安全人员专注于更复杂的逻辑判断和未知漏洞挖掘。

对于安全团队而言，Pentagi 的价值在于：

  • 降低渗透测试门槛：非专业安全人员也能执行标准化测试
  • 提升效率：多 Agent 并行处理，缩短测试周期
  • 标准化输出：统一的报告格式，便于审计与对比

不过需要注意的是，AI Agent 在安全领域的应用仍有局限性，尤其是在面对未知漏洞、复杂业务逻辑和强对抗环境时，纯 AI 驱动的渗透测试尚无法完全替代人工专家。
