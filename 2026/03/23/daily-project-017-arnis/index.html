arnis：将真实世界带入 Minecraft 的开源神器

项目概览

arnis（louis-e/arnis）是一款将真实世界地理位置高保真映射到 Minecraft 的开源工具。无论你想重建巴黎街头、珠峰山体，还是自家小区，arnis 都能将真实地图数据转化为 Minecraft 中的方块世界，支持极高的地形细节还原。

GitHub：https://github.com/louis-e/arnis

技术原理

arnis 的核心流程是 真实地图数据 → 地形解析 → Minecraft 方块映射：

简单来说，它把真实世界的经纬度坐标对应到 Minecraft 的 X/Z 坐标，用高程数据（DEM）还原三维地形，再基于地表类型（植被、水体、建筑区）将不同的真实地物分类映射为对应的 Minecraft 方块——比如草地、石头、水、树木等。

支持的真实地图数据源

arnis 之所以能实现高保真还原，离不开丰富的真实地图数据源支持：

数据源 类型 说明
OpenStreetMap 矢量地图 提供建筑、道路、河流等语义数据
SRTM / Copernicus DEM 高程数据 30m/90m 分辨率的地形高程
ESRI Satellite 卫星影像 用于地表分类（植被、水体、不透水面）
Bing Maps 卫星/街道图 补充高分辨率影像来源

多源数据融合是 arnis 的核心能力：卫星影像提供地表覆盖类型，DEM 提供三维地形，OSM 提供人工地物（建筑、道路），三者叠加才能做到”不只是山，还原山上的树和房子”。

生成效果对比

以纽约中央公园为例：

真实地图 Minecraft 生成效果
卫星图：绿色植被覆盖，轮廓清晰 草地 + 深色树木方块群，还原公园形态
高程：地势略有起伏 地形高度还原，草坪区域有自然起伏
OSM：周边建筑群 边界用石砖/深色方块填充建筑轮廓

效果评价： 山体、大面积水域、平原等自然地形还原度极高；城市区域因 OSM 建筑数据精度不同，还原效果有差异。整体在视觉和地理意义上都具备相当高的可信度。

技术亮点

  1. 多源数据融合架构
    不依赖单一数据源，卫星影像 + DEM + OSM 矢量数据叠加，分层处理后融合输出，保证信息完整性。

  2. 高度可控的生成粒度
    支持按坐标范围（经纬度框选）、按城市名称、按兴趣点（POI）周边半径等多种方式指定生成区域，灵活度高。

  3. Minecraft 版本兼容
    输出格式支持 .schematic（JE 1.12-1.17 主流）和 .nbt（新版格式），覆盖大多数服务器和单人存档场景。

  4. 自动化地表分类
    利用卫星影像光谱特征自动区分：水域、植被、不透水面、裸土，无需人工标注，大幅降低使用门槛。

  5. CLI 优先，开箱即用
    纯命令行工具，一条命令指定坐标和范围即可开始生成，无复杂 GUI 学习成本，适合集成到自动化脚本。

Rayon 是 Rust 的数据并行库，par_iter() 让多个 CPU 核心同时计算不同的 flood fill 区域，充分利用多核。

总结

arnis 是目前 GitHub 上将真实世界映射到 Minecraft 领域最活跃的项目之一。它用工程化的方式将地理信息系统的数据处理流程，优雅地封装成了任何人都能使用的工具。对地理信息、游戏开发、城市规划可视化和 Minecraft 服务器运营者来说，这都是一个值得关注的开源项目。

如果你想把自己的家乡、喜欢的城市、甚至虚构世界的参考地景搬进 Minecraft，arnis 是目前门槛最低、效果最好的开源方案。
