everything-claude-code：让 AI Coding Agent 拥有「肌肉记忆」

发表于2026-03-23|更新于2026-03-28|开源项目
|浏览量:

项目概览

everything-claude-code — affaan-m/everything-claude-code ⭐ 99,707

又是一个 star 接近 10 万的超热门项目。它解决的问题非常直接：原生 Claude Code 太「傻」，缺乏长期记忆、领域直觉和安全意识，而这个项目试图给它装上一套完整的「神经系统」。

该项目自称是 Agent Harness Performance Optimization System，面向 Claude Code、Codex、Opencode、Cursor 等主流 AI Coding Agent，通过模块化的 Skills / Instincts / Memory / Security 架构，让 Agent 在真实项目中表现得更像一位经验丰富的老开发者。

核心设计：四大模块拆解

1. 🧠 Skills — 可组合的能力单元

Skills 是项目最核心的设计理念之一。它将 Agent 的能力拆解为可插拔、可复用的技能单元，每个 Skill 包含：

  • 触发条件（何时激活）
  • 执行逻辑（具体怎么做）
  • 验收标准（如何判断成功）

类似于 LangChain 的 Tool 概念，但更贴近 IDE 实际场景。例如一个 git-safe-commit Skill，会在检测到敏感数据变更时自动触发告警而非直接提交。

2. ⚡ Instincts — 本能反应层

Instincts 是我认为最有意思的设计。它模拟人类的直觉反应：

不需要每次都从头分析，Agent 看到特定模式就直接「肌肉记忆」式响应。

比如：

  • 看到 package.json 变更 → 自动运行 npm audit
  • 看到数据库迁移文件 → 提示备份当前 DB
  • 看到 *.env 文件被修改 → 触发 security review

这套机制大幅降低了 Agent 在重复决策上的 token 消耗，同时提高了响应质量。

3. 💾 Memory — 跨会话持久记忆

原生 Claude Code 是无状态的，每次对话都是从零开始。everything-claude-code 引入了分层记忆系统：

记忆层级 存储内容 生命周期
Short-term 当前会话上下文 会话内
Medium-term 最近 N 个项目的决策历史 数天
Long-term 架构决策、安全模式、团队规范 永久

通过向量数据库 + 结构化存储的混合方案，Agent 能在新会话中「想起来」之前做过的技术决策。

4. 🔒 Security — 内置安全防线

安全模块包含：

  • 静态扫描：检测硬编码密钥、SQL 注入风险、不安全依赖
  • 动态监控：实时代码变更风险评估
  • 审计日志：所有敏感操作完整记录可追溯

与原生 Claude Code 的 Careful 模式相比，everything-claude-code 的 Security 层更主动，不是等 Agent 问「是否继续」，而是在执行前就拦截高风险操作。

与 Claude Code 原生能力对比

能力维度 Claude Code 原生 everything-claude-code
上下文记忆 ❌ 无 ✅ 三层持久记忆
领域直觉 ❌ 无 ✅ Instincts 模式匹配
安全审查 ⚠️ 基础 ✅ 主动拦截 + 审计
技能复用 ❌ 无 ✅ Skills 生态
配置成本 低 中高（需要维护 Skills 库）

结论：原生适合快速探索，原生增强版适合严肃工程。

核心原理分析

1. Skills 的事件驱动激活机制

everything-claude-code 的 Skills 并非简单配置，而是一套事件驱动的激活系统。每个 Skill 的触发由 Claude Code 的 Hook 机制（PreToolUse / PostToolUse / Stop 等事件）驱动：

事件匹配器（matcher） 是 Skills 激活的核心：

  • 它是一个布尔表达式，匹配 Claude Code 内部事件流
  • 匹配成功时触发对应 Hook，执行 Skills 定义的操作
  • 这种设计让 Skills 的激活完全声明式，无需在业务代码中手动调用

2. Instincts 的模式匹配引擎

Instincts（本能反应）是该项目最有深度的设计之一。它基于模式 → 动作的规则引擎：

置信度机制：每个 Instinct 有 0~1 的置信度，当 Agent 执行高风险操作时，只有高置信度 Instinct 才会被自动触发；中等置信度 Instinct 给出提示而非自动执行。

3. 三层记忆架构

记忆系统的分层设计解决了”短上下文 vs 长期知识”的矛盾：

会话开始时（SessionStart Hook），Medium → Short 的记忆被加载到上下文；会话结束时（SessionEnd Hook），Short → Medium 的记忆被持久化。Long-term 记忆通过向量检索按需召回。

4. Agent 分层委托架构

每个 Agent 是一个 Markdown 文件（agents/planner.md），包含：

  • name / description — Agent 元数据
  • tools — Agent 可使用的工具范围（最小权限原则）
  • model — 指定使用的模型（如 planner 用 Opus，代码生成用 Sonnet）
  • --- 分隔后的 system prompt 正文

5. 规则（Rules）的分层继承

规则按”通用 → 语言特定”分层，Claude Code 加载时会合并而非替换——语言特定规则继承 common 的所有规则，并在冲突时以语言特定规则优先。

源码实现深度解析

1. 插件清单与组件注册

组件路径 glob 模式是 Claude Code 插件系统的标准发现机制：插件启动时扫描 components 下所有文件并注册。skills/*/SKILL.md 约定俗成让每个 Skill 自身成为一个子目录（包含 SKILL.md + 资源文件），避免单文件 Skill 的资源组织困境。

2. Hook 脚本的 Session Persistence

Session Persistence 的触发点：Stop 事件（Claude Code 等待用户下一条输入时）触发 SessionEnd Hook，确保每次交互间隙都保存记忆，不会因会话超时丢失数据。

3. Instinct 的 YAML 定义格式

YAML → 代码的加载逻辑：

  1. evolve_threshold 机制让高频 Instinct 自动”毕业”为正式 Skill（通过 /evolve 命令）
  2. tools_blocked 字段在 Hook 层面拦截工具调用，而非仅给出警告
  3. examples 用于在 Agent 的 system prompt 中注入零样本上下文

4. 多语言规则的条件加载

按需安装而非全量复制：避免 Agent 在处理 TypeScript 项目时被迫阅读 Go 的规则文件，减少上下文噪声。同时 common/ 规则始终存在，确保跨语言项目（如前后端同仓库）也能得到通用规范约束。

5. NanoClaw 会话管理核心

会话压缩（Compact） 是解决 Claude Code 长会话上下文爆炸的核心机制：当 transcript 接近上下文窗口上限时，将历史交互压缩为摘要，保留关键决策和代码变更摘要，同时丢弃中间的问题探索过程。

技术亮点

  1. Event-Driven Skill Activation：基于事件总线解耦 Skills，扩展性强
  2. Hybrid Memory Store：向量检索 + 键值存储，平衡速度与准确性
  3. Zero-Config Security Boundary：开箱即用的安全基线，无需逐条配置
  4. Tree-of-Thought Memory：记忆以决策树形式组织，方便追溯和剪枝

总结

everything-claude-code 的核心价值在于把 AI Coding Agent 从「每次都是新手」提升到「积累经验的老手」。它的 Skills/Instincts/Memory/Security 四件套设计，为构建企业级 AI 开发助手提供了可复用的架构参考。

10 万 star 不是偶然——它精准击中了当前 AI Coding Agent 在工程化场景中的几个核心痛点。如果你正在基于 Claude Code 构建内部开发平台，这个项目的设计思路值得深入研究。
