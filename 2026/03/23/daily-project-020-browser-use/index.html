browser-use：让 AI 操控浏览器的开源框架

项目概览

browser-use 是当前 GitHub 趋势热门项目，旨在让 AI 代理（Agent）能够直接操控浏览器完成在线任务。它将自然语言指令转化为真实的浏览器操作，实现”AI 代替人类上网”的愿景。

  • GitHub: browser-use/browser-use
  • 语言: Python
  • Star: 趋势热门

解决的问题

传统浏览器自动化工具（如 Selenium、Playwright）需要开发者手动编写操作指令，门槛高、效率低。browser-use 通过接入大语言模型（LLM），让 AI 自动分析页面、规划操作步骤并执行，真正做到”说一句话，AI替你上网”。

核心原理分析

browser-use 的核心设计理念是将 Web 页面建模为一个可供 LLM 操作的工具集，而不是简单地暴露原始 DOM。系统通过多层抽象，将浏览器状态转化为 LLM 可理解的语义表示，再将 LLM 的输出（结构化动作）转化为真实浏览器操作。整个闭环构成了一个观察-推理-行动（OODA）循环的自动化执行体。

DOM 抽象层：ElementLocator 与 DOMState

browser-use 没有直接暴露原始 HTML，而是通过 DOMState 对页面进行结构化建模。每个可交互元素被建模为 DOMInteractedElement，包含：

  • element_id：元素在当前页面中的唯一索引
  • xpath：元素的 XPath 路径（用于精确定位）
  • dom_path：DOM 树中的层级路径
  • parent_id、children_ids：父子关系链
  • tag_name、attributes：元素元数据
  • text_content、input_value：内容信息

这种抽象使 LLM 只需关注”我要点击第几个按钮”这类逻辑操作，而无需理解底层 DOM 实现细节。

双通道感知：视觉 + DOM

browser-use 的 Agent 在每次决策前收集两类信息：

  1. DOM 快照（get_dom_state）：提取当前页面所有可交互元素的结构化描述，注入到 LLM 的 prompt 中
  2. 截图（screenshot）：当 use_vision=True 时，截图作为图片 token 一起发送给多模态模型

这种”视觉 + DOM”双通道设计弥补了纯 DOM 理解的歧义问题——同一个按钮在 DOM 中可能只是一行 <button id="submit">，但截图能告诉 LLM 它在页面上的实际视觉位置和大小。

技术亮点

1. 多模型支持

支持 OpenAI、Anthropic、Azure OpenAI、Google Gemini 等主流 LLM 接口，灵活切换。

2. 视觉+DOM 双通道

结合页面截图（视觉）和 DOM 结构（文本），让 LLM 准确理解页面布局，避免纯 DOM 理解的歧义。

3. 任务记忆机制

Agent 维护对话历史和页面状态上下文，支持多轮交互完成复杂任务（如”先登录，再搜索，再下载”）。

4. 可扩展性强

  • 支持自定义工具（Tools）
  • 支持流式输出（Streaming）
  • 支持本地浏览器或无头模式（Headless）

5. 错误恢复

操作失败时自动重试，结合 LLM 自我纠错能力，提升任务成功率。

总结

browser-use 代表了 AI Agent 在 Web 自动化领域的重要突破。它将复杂的浏览器操控封装为自然语言交互，大大降低了 AI 上网的技术门槛。随着多模态模型的成熟，”AI 代替人类完成一切在线任务”正逐步变为现实。

如果你想快速实现 AI 驱动的浏览器自动化，browser-use 是一个值得深入研究的开源项目。
