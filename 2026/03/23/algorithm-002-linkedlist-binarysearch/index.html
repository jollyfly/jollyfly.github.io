题目六：两数相加（LeetCode 2）

题目描述

给你两个非空链表，表示两个非负整数。数字按逆序存储，每个节点一位。将其相加并返回一个新的链表。

示例：l1 = [2,4,3]，l2 = [5,6,4] → 返回 [7,0,8]（即 342 + 465 = 807）

分类标签

维度 标签
数据结构 链表
算法思想 模拟、数学
难度 中等

初步思路（暴力）

思路 1：转整数再相加（会溢出）

⚠️ 问题：整数可能超出 long 范围（如链表长度 50+），直接崩溃。

思路 2：按位模拟（正确思路的雏形）

逐节点相加，记录进位，生成新节点。

画图演示：进位传播过程

        C0["carry = 0"]
    end

    subgraph Step2["第2位相加: 4 + 6 + 0 = 10, 进位=1"]
        N2["新节点: 0"]
        C1["carry = 1"]
    end

    subgraph Step3["第3位相加: 3 + 4 + 1 = 8, 进位=0"]
        N3["新节点: 8"]

    end

    subgraph 结果
        R["7 → 0 → 8 → NULL"]
    end

    result->>result: 7 → 0 → 8

优化过程

优化点：不需要转整数，原位按位模拟，时间 O(max(m,n))，空间 O(1)（不含结果链表）。

最优解

使用**哑节点（dummy）**简化头节点处理，遍历两链表直到均到达末尾且无进位。

题目七：删除链表的第 N 个节点（LeetCode 19）

题目描述

给你一个链表，删除链表的倒数第 N 个节点，返回头节点。

示例：[1,2,3,4,5]，n=2 → [1,2,3,5]

分类标签

维度 标签
数据结构 链表
算法思想 双指针
难度 中等

初步思路（暴力）

  1. 第一次遍历：统计链表长度 len
  2. 第二次遍历：走到第 len - n 个节点，删除它

缺点：需要两趟遍历，且要处理 n == len（删头节点）的边界

画图演示：快慢指针删除步骤

    flowchart LR
    subgraph 初始状态
        H["1"] --> A["2"] --> B["3"] --> C["4"] --> D["5"] --> N["NULL"]
        N2["n = 2"]
    end

    subgraph 快指针先走n步
        F1["fast → 3"] 
        S1["slow → 1"]
    end

    subgraph 快慢同时走直到fast到底
        F2["fast → NULL"] 
        S2["slow → 3"]
    end

    subgraph 删除slow.next
        D3["删除节点4"]
        S3["slow → 3"] --> E["5"]
    end
  
    sequenceDiagram
    participant H as 头节点
    participant F as fast指针
    participant S as slow指针
    participant Del as 待删节点

    Note over H,F,S,Del: 链表: 1→2→3→4→5, n=2
    H->>F: fast 先走 3 步
    F->>Del: fast 指向节点3
    S->>H: slow 从头开始（节点1）
    loop 同步后移
        F->>F: fast = fast.next
        S->>S: slow = slow.next
    end
    Note over S: slow 指向节点3
    S->>Del: 记录 slow.next = 节点4
    Del->>Del: slow.next = slow.next.next
    Note over H: 结果: 1→2→3→5

最优解：快慢指针（一趟）

设置哑节点 dummy，快指针先走 n+1 步，然后快慢同时走。当快指针到达末尾时，慢指针正好在待删节点前一个位置。

题目八：搜索旋转排序数组（LeetCode 33）

题目描述

整数数组 nums 升序排列，但被旋转了（将尾部若干元素移到开头）。给定目标值 target，返回其下标，不存在返回 -1。

示例：nums = [4,5,6,7,0,1,2]，target = 0 → 返回 4

分类标签

维度 标签
数据结构 数组
算法思想 二分查找
难度 中等

初步思路（暴力）

遍历数组找目标，时间 O(n)，但没有利用旋转数组仍有两段升序的特性。

画图演示：旋转排序数组 + 二分边界判断

    flowchart TD
    subgraph 旋转数组结构
        L["[4, 5, 6, 7 | 0, 1, 2]"]
        P["旋转点"]
    end

    subgraph 二分判断逻辑
        M["mid = (left + right) / 2"]
        C1{"nums[mid] == target?"}
        C2{"nums[left] <= nums[mid]?<br/>（左半段有序）"}
        C3{"target 在 [nums[left], nums[mid])?"}
        C4{"target 在 [nums[mid+1], nums[right])?"}
        R1["right = mid - 1"]
        R2["left = mid + 1"]
        R3["答案: mid"]
        R4["答案: -1"]
    end
  
    flowchart LR
    A["target = 0"] --> B{"mid=3, nums[3]=7<br/>7>0, 在左半段?"}
    B -->|否| C{"nums[3] >= nums[0]?<br/>7>=4 是，左半段有序"}
    C -->|是| D{"target >= nums[0]?<br/>0>=4 否"}
    D -->|否| E["target 在右半段<br/>left = mid+1 = 4"]
    E --> F{"mid=4, nums[4]=0<br/>找到!"}
    F --> G["return 4"]

最优解

二分查找变体：每次确定 target 落在哪一段有序区间内，缩小搜索范围。

Java 实现

复杂度分析

指标 复杂度 说明
时间 O(log n) 二分查找
空间 O(1) 原地操作

题目九：查找元素的第一个和最后一个位置（LeetCode 34）

题目描述

给定升序数组 nums，找目标值 target 的第一个和最后一个下标。不存在返回 [-1, -1]。

示例：nums = [5,7,7,8,8,10]，target = 8 → [3, 4]

分类标签

维度 标签
数据结构 数组
算法思想 二分查找
难度 中等

初步思路（暴力）

遍历数组，记录第一个和最后一个出现位置，时间 O(n)。

画图演示：左右边界收缩过程

    flowchart TD
    subgraph 初始
        A["nums = [5, 7, 7, 8, 8, 10]"]
        T["target = 8"]
    end

    subgraph 找左边界
        L1["left=0, right=5, mid=2, nums[2]=7 < 8 → left=3"]
        L2["left=3, right=5, mid=4, nums[4]=8 == 8 → left收缩 → left=4"]
        L3["left=4, right=5, mid=4 → left=5, 退出 → leftBound=4"]
    end

    subgraph 找右边界
        R1["left=0, right=5, mid=2, nums[2]=7 < 8 → left=3"]
        R2["left=3, right=5, mid=4, nums[4]=8 == 8 → right收缩 → right=3"]
        R3["left=3, right=3, mid=3, nums[3]=8 → right=2, 退出 → rightBound=4"]
    end

    subgraph 结果
        RES["[leftBound, rightBound] = [4, 4]"]
    end
  
    sequenceDiagram
    participant A as 数组
    participant LB as 找左边界
    participant RB as 找右边界

    Note over A: nums = [5, 7, 7, 8, 8, 10], target = 8
    par 找左边界
        LB->>LB: left=0, right=5
        LB->>LB: mid=2 → 7<8 → left=3
        LB->>LB: left=3, right=5 → mid=4 → 8==8 → continue
        LB->>LB: left=4, right=5 → mid=4 → left=5
        LB->>LB: left=5, right=5 → mid=4 → left=5
        LB->>LB: left=5 > right=5 退出
        LB->>LB: leftBound = 4
    and 找右边界
        RB->>RB: left=0, right=5
        RB->>RB: mid=2 → 7<8 → left=3
        RB->>RB: left=3, right=5 → mid=4 → 8==8 → continue
        RB->>RB: left=3, right=4 → mid=3 → 8==8 → continue
        RB->>RB: left=3, right=2 退出
        RB->>RB: rightBound = 4
    end
    Note over A: 结果: [4, 4]

最优解

分别用二分查找找左边界和右边界（左边界：target <= nums[mid] 时收缩 right；右边界：target >= nums[mid] 时收缩 left）。

Java 实现

复杂度分析

指标 复杂度 说明
时间 O(log n) 两次二分
空间 O(1) 原地操作

题目十：寻找两个正序数组的中位数（LeetCode 4）

题目描述

给定两个正序（升序）数组 nums1、nums2，返回它们合并后的中位数。

要求：时间复杂度 O(log(m+n))。

示例：nums1 = [1,3]，nums2 = [2] → 返回 2.0

分类标签

维度 标签
数据结构 数组
算法思想 二分查找
难度 困难

初步思路（暴力）

合并两数组后排序，时间 O((m+n) log(m+n))，但远达不到 O(log(m+n)) 要求。

画图演示：分割线过程

    flowchart TD
    subgraph 两数组
        N1["nums1: [1, 3 | 4, 5]"]
        N2["nums2: [2 | 6, 7]"]
    end

    subgraph 分割线
        P1["左半部分所有元素 ≤ 右半部分所有元素"]
        S["i = 1（nums1左半长度）"]
        P2["nums1[i-1]=1, nums1[i]=3"]
        P3["nums2[j-1]=2, nums2[j]=6"]
    end

    subgraph 满足条件
        C1["1 ≤ 2 ✓<br/>3 ≤ 6 ✓"]
    end

    subgraph 中位数计算
        M{"总长度为偶数?"}
        M1["(maxLeft + minRight) / 2"]
        M2["maxLeft （奇数时唯一中位数）"]
    end

    subgraph 结果
        R["中位数 = (2 + 3) / 2 = 2.5"]
    end

最优解

二分查找分割线：将找中位数问题转化为找第 k 小元素或满足条件的分割线。利用 log(m+n) 的二分思想。

Java 实现

复杂度分析

指标 复杂度 说明
时间 O(log(min(m, n))) 在较短数组上二分
空间 O(1) 原地操作

总结

题号 题目 核心技巧 时间 空间
6 两数相加 链表按位模拟 + 进位传播 O(max(m,n)) O(max(m,n))
7 删除倒数第 N 节点 快慢指针（间距 n+1） O(L) O(1)
8 搜索旋转数组 二分查找（判断有序区间） O(log n) O(1)
9 查找左右边界 二分左右边界分别收缩 O(log n) O(1)
10 两个正序数组中位数 二分分割线 O(log(min(m,n))) O(1)

核心规律：

  • 链表问题：哑节点简化边界，哨兵统一处理
  • 二分查找：先写普通版，再改造成变体（旋转、左边界、右边界、分割线）
  • 进位/指针：画图演示每一步状态变化，比文字描述直观十倍
