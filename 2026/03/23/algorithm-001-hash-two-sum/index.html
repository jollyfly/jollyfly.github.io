题目一：两数之和（LeetCode 1）

题目描述

给定一个整数数组 nums 和一个整数 target，请你在该数组中找出和为目标值 target 的两个整数的下标，并返回它们的数组下标。
你可以假设每种输入只会对应一个答案，且同样的元素不能被重复利用。

示例：

分类标签

数组 哈希表

初步思路（暴力解）

两层 for 循环，枚举所有 (i, j) 配对，检查 nums[i] + nums[j] == target。

代码实现：

两层循环，最坏情况遍历 $\frac{n(n-1)}{2}$ 次，时间复杂度 $O(n^2)$，空间复杂度 $O(1)$。

优化过程

暴力解的瓶颈在于：找补数 需要 $O(n)$ 遍历。我们希望把”找补数”优化到 $O(1)$，自然想到哈希表：

  • 遍历数组时，以 数值 → 下标 存入 HashMap
  • 对于当前元素 nums[i]，查 target - nums[i] 是否已在 Map 中
  • 找到了就返回；没找到就把当前元素加入 Map

一次遍历搞定，时间从 $O(n^2)$ 降到 $O(n)$。

    subgraph 遍历过程
        A1["i=0<br/>nums[0]=2<br/>补数=9-2=7"] --> B1{"7 在 HashMap 中?"}
        B1 -->|否| C1["存入 Map<br/>{2 → 0}"]
        C1 --> A2["i=1<br/>nums[1]=7<br/>补数=9-7=2"]
        A2 --> B2{"2 在 HashMap 中?"}
        B2 -->|是!| D["返回 [0, 1] ✅"]
    end

复杂度分析

方案 时间复杂度 空间复杂度
暴力双重循环 $O(n^2)$ $O(1)$
HashMap 最优解 $O(n)$ $O(n)$

题目二：无重复字符的最长子串（LeetCode 3）

题目描述

给定一个字符串 s，请你找出其中不含有重复字符的最长子串的长度。

示例：

分类标签

滑动窗口 哈希表 双指针

初步思路（暴力解）

枚举所有子串起点，再向后扩展并检查重复，复杂度 $O(n^3)$，不可接受。

优化过程

滑动窗口是本题的核心：

  • 维护一个左指针 left 和右指针 right 构成的窗口
  • 右指针不断右移，将字符加入窗口
  • 若出现重复字符，则左指针右移收缩窗口，直到重复字符被移除
  • 全程用 HashMap<字符, 下标> 记录每个字符最近出现的位置

当右指针遍历完整个字符串时，记录的最大窗口长度即为答案。

算法图解

以 s = "abcabcbb" 为例，展示窗口扩张与收缩过程：

    subgraph step1["Step 1-3: 窗口扩张 [a,b,c]"]
        R1["right=0, char=a"] --> R2["right=1, char=b"]

        R3 --> L1["无重复，最大长度=3"]
    end

    subgraph step2["Step 4-6: 遭遇重复 'a'"]

        R5 -->|"b已存在"| S2["left右移到b的下一位"]
        S2 --> R6["right=5, char=c"]

    subgraph step3["Step 7-8: 继续扩张]
        R7["right=6, char=b"] -->|"b已存在"| S3["left右移"]
        R8["right=7, char=b"] -->|"b已存在"| S4["left右移"]
    end

最优解 Java 实现

关键点：left = Math.max(left, map.get(c) + 1) 确保 left 不会回退，处理了 “abba” 这类边界情况。

复杂度分析

方案 时间复杂度 空间复杂度
暴力枚举子串 $O(n^3)$ $O(min(m, n))$
滑动窗口 + HashMap $O(n)$ $O(min(m, n))$

题目三：LRU 缓存（LeetCode 146）

题目描述

请你设计并实现一个满足 LRU（最近最少使用）缓存约束的数据结构。
LRUCache 类：

  • LRUCache(int capacity) — 初始化缓存容量
  • int get(int key) — 获取值（不存在返回 -1），同时将该键值对标记为最近使用
  • void put(int key, int value) — 插入或更新值，若超过容量则淘汰最久未使用的项

示例：

分类标签

设计 哈希表 双向链表

核心思路

LRU 需要两种操作都在 $O(1)$ 内完成：

  • get：O(1) 查找 + O(1) 移动到头部
  • put：O(1) 插入/更新 + O(1) 淘汰尾部

数据结构组合：

  • HashMap<Integer, Node>：O(1) 按 key 找到 Node
  • 双向链表：O(1) 在头部插入（最新使用），O(1) 在尾部删除（最久未使用）

    end

    subgraph get操作["get(1) 后：Node(k1,v1) 移到头部"]
        H2["head"] <--> N1b["Node(k1,v1) ✅"] <--> N2b["Node(k2,v2)"] <--> T2["tail"]

最优解 Java 实现

设计亮点：

  • 虚拟头尾节点（dummy head/tail）避免空指针判断
  • moveToHead 实现 get 时的”最近使用”标记
  • removeTail 实现 put 时的 LRU 淘汰

题目四：合并区间（LeetCode 56）

题目描述

给出一个区间的集合，合并所有重叠的区间。

示例：

分类标签

数组 排序 双指针

初步思路（暴力解）

遍历每对区间检查是否重叠，复杂度 $O(n^2)$，且难以处理多区间级联合并的情况。

优化过程

排序 + 单次遍历是本题的最优解：

  1. 按起点排序：所有可能重叠的区间在排序后会相邻
  2. 贪心合并：依次检查相邻区间是否能合并

合并规则（当前区间 curr，结果栈顶 last）：

  • 若 curr.start <= last.end，则重叠，合并：last.end = max(last.end, curr.end)
  • 否则不重叠，直接将 curr 加入结果

算法图解

以 intervals = [[1,3],[2,6],[8,10],[15,18]] 为例：

    graph TD

    C --> D["curr=[2,6]<br/>start=2 ≤ last.end=3，重叠！<br/>last.end = max(3,6)=6<br/>合并为 [1,6]"]
    D --> E["curr=[8,10]<br/>start=8 > last.end=6，不重叠<br/>加入结果: [[1,6]]"]

方案 时间复杂度 空间复杂度

题目五：最小覆盖子串（LeetCode 76）

题目描述

给你一个字符串 s 和一个字符串 t，请你在 s 中找出包含 t 中所有字符的最小子串。
若 s 中不存在这样的子串，返回空字符串 ""。

示例：

分类标签

滑动窗口 哈希表 双指针 字符串

初步思路（暴力解）

枚举所有子串起点，对每个起点向后扩展直到覆盖 t，再取最小，复杂度 $O(n^2 \cdot |t|)$，完全不可行。

优化过程

滑动窗口 + 计数器：

  1. 维护一个窗口 [left, right)，用 need 和 window 两个 HashMap 统计字符出现次数
  2. right 不断右移扩大窗口，直到窗口恰好覆盖 t（valid == need.size()）
  3. 记录当前最小窗口后，left 开始右移收缩，试图找更小的覆盖子串
  4. 重复以上过程直到 right 遍历完

关键点：

  • valid 计数器记录当前窗口中已满足”每种字符出现次数 ≥ t 中该字符出现次数”的字符种类数
  • 收缩 left 时，若收缩后不再满足覆盖条件，valid 会减小，right 会继续扩张寻找下一个候选窗口

算法图解

以 s = "ADOBECODEBANC"，t = "ABC" 为例：

    subgraph 扩张阶段["right 右移扩张窗口"]

最优解 Java 实现

核心技巧：

  • valid 计数器让”收缩”条件判断从遍历整个 HashMap 的 $O(m)$ 降到 $O(1)$
  • needKind 统计 t 中不同字符种类，避免每次都用 containsValue 判断是否完全覆盖

复杂度分析

方案 时间复杂度 空间复杂度
暴力枚举子串 $O(n^2 \cdot |t|)$ $O(|t|)$
滑动窗口 $O(n)$ $O(1)$（ASCII 字符集大小固定为 128）

总结

题目 核心技巧 时间 空间
两数之和 HashMap 缓存补数 $O(n)$ $O(n)$
无重复字符最长子串 滑动窗口 + 记录最近位置 $O(n)$ $O(1)$
LRU 缓存 HashMap + 双向链表 $O(1)$ 均摊 $O(c)$
合并区间 排序 + 贪心合并 $O(n \log n)$ $O(n)$
最小覆盖子串 滑动窗口 + valid 计数器 $O(n)$ $O(1)$

通用套路：

  • 看到 $O(n^2)$ → 考虑哈希表降到 $O(n)$
  • 看到子串/子数组问题 → 考虑滑动窗口双指针
  • 需要 $O(1)$ 访问 + $O(1)$ 淘汰 → HashMap + 双向链表
  • 区间重叠问题 → 排序后贪心
