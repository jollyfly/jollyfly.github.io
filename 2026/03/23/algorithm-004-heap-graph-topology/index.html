题目十六：合并 K 个有序链表（LeetCode 23）

题目描述

给定 K 个有序链表，将其合并为一个有序链表，返回头结点。

示例：

分类标签

堆 优先队列 链表 分治 归并

初步思路（暴力）

依次遍历 K 条链表的所有节点，将值全部存入数组，排序后重新构建链表。

问题： 需遍历全部 N 个节点，时间复杂度 O(N)，但排序 O(N log N) 成为瓶颈。

    subgraph "Step 1：初始小根堆"

    subgraph "Step 2：弹出 1(L1)，插入 4(L1)"
        H2["heap: [1(L2), 2(L3), 4(L1)]"]
        style H2 fill:#ccffcc
    end

    subgraph "Step 3：弹出 1(L2)，插入 3(L2)"
        H3["heap: [2(L3), 3(L2), 4(L1)]"]
        style H3 fill:#ccffcc
    end

    subgraph "Step 4：弹出 2(L3)，插入 6(L3)"
        H4["heap: [3(L2), 4(L1), 6(L3)]"]

优化过程

方案 思路 时间 空间
暴力排序 全量收集 + 排序 O(N log N) O(N)
两两合并 依次合并 K-1 次 O(KN) O(1)
分治合并 两两归并，递归 log K 层 O(N log K) O(log K)
优先队列 每次弹出最小值 O(N log K) O(K)

优先队列（最小堆）始终维护 K 个链表头的最小值，每次 pop O(log K)，共 N 次弹出。

最优解

采用优先队列（小根堆），始终保持堆顶为当前最小值：

  1. 将所有链表头节点加入小根堆
  2. 弹出堆顶（当前最小），接入结果链表
  3. 若弹出的节点有 next，则 next 入堆
  4. 重复直到堆空

复杂度分析

维度 复杂度 说明
时间 O(N log K) N = 总节点数，K = 链表数；每次 heap.push/pop = O(log K)，共 N 次
空间 O(K) 堆中最多存 K 个节点（每条链表一个头）
适用场景 K 大、N 极大时，相比 O(N log N) 有明显优势

题目十七：数据流的中位数（LeetCode 295）

题目描述

数据流中不断流入整数，需要随时返回当前已见数据的中位数。

分类标签

堆 双堆 数据流 设计 排序

初步思路（暴力）

每来一个数就插入数组并排序，O(N) 插入；找中位数 O(1)。

问题： 每次插入都 O(N)，N 次插入总 O(N²)。

优化过程

方案 思路 addNum findMedian 总时间
暴力数组排序 插入排序 O(N) O(1) O(N²)
BST / TreeSet 有序结构 O(log N) O(log N) O(N log N)
双堆 一个大根堆 + 一个小根堆 O(log N) O(1) O(N log N)

双堆核心思想：

  • 左堆（大根堆）： 维护较小的一半数，堆顶是左半部分最大值
  • 右堆（小根堆）： 维护较大的一半数，堆顶是右半部分最小值
  • 平衡条件： 左堆大小 == 右堆大小 或 左堆大小 = 右堆大小 + 1（奇数时左堆多一个）

最优解

复杂度分析

维度 复杂度 说明
时间 addNum O(log N) 堆的 push/pop 为对数级别
时间 findMedian O(1) 直接读取两个堆顶
空间 O(N) 所有数字分布在两个堆中
核心技巧 左堆 size = 右堆 size 或 +1，保证中位数在两个堆顶之间

题目十八：序列化与反序列化二叉树（LeetCode 297）

题目描述

设计一种算法，将二叉树序列化为字符串（可以传输/存储），并根据字符串反序列化重建原二叉树。

分类标签

树 BFS DFS 序列化 设计 层序遍历

初步思路（暴力）

前序遍历递归序列化，遇见 null 就输出特殊标记。但需要处理边界（空树）。

问题： 反序列化时难以区分 “null” 和真实值 0；且空树需特殊处理。

画图演示（序列化树结构）

前序遍历序列化过程：

    style L1 fill:#ffcccc
    style R1rl fill:#ffcccc
    style R1rr fill:#ffcccc
    style R1r fill:#ccffcc

    subgraph "序列化路径"
        SEQ["1 → 2 → null → null → 3 → 4 → null → null → 5 → null → null"]
        style SEQ fill:#bbddee
    end

BFS 层序序列化：

    graph TD
    subgraph "层序遍历队列状态"
        Q1["队列: [1]"]
        Q2["队列: [2, 3]"]
        Q3["队列: [null, 4, 5]"]
        Q4["队列: [4, 5, null, null]"]
        Q5["队列: [5, null, null, null]"]
    end

    Q1 --> Q2 --> Q3 --> Q4 --> Q5

优化过程

方案 编码方式 反序列化难度 效率
前序 DFS 递归，null 占位 需预读或队列 中
BFS 层序 队列，null 也入队 简单（队列顺序一致） 高
括号表示 [root,left,right] 需解析 低

BFS 层序方案：每个节点依次出队，其左右子节点依次入队（null 也入），队列空时停止。读字符串时按顺序即可重建。

最优解

采用 BFS 层序遍历序列化与反序列化，代码简洁且重建天然匹配：

复杂度分析

维度 复杂度 说明
时间 O(N) 每个节点访问常数次
空间 O(N) 队列 + 字符串存储
关键点 序列化时 null 也入队，保证反序列化顺序与队列完全一致
可读性 BFS 字符串比 DFS 字符串更紧凑（null 更少）

题目十九：字符串的排列（LeetCode 567）

题目描述

给定两个字符串 s1 和 s2，判断 s2 是否包含 s1 的任意排列（即 s1 的字符重排列），返回 true/false。

分类标签

滑动窗口 哈希表 字符计数 双指针 数组

初步思路（暴力）

枚举 s2 中所有长度为 |s1| 的子串，对每个子串排序后与 s1 排序结果比较。

问题： 排序 O(|s1| log |s1|)，在 s2 上滑动 |s1| 次，总复杂度 O(|s2| × |s1| log |s1|)。

画图演示（滑动窗口滑动过程）

字符计数窗口变化（s1 = “ab”, s2 = “eidbaooo”）：

    graph LR
    subgraph "窗口位置 i=0: 'eid'"
        W1["窗口: e i d\ncnt: {a:0,b:0,e:1,i:1,d:1}\ntarget: {a:1,b:1}"]
        style W1 fill:#ffcccc
    end

    subgraph "窗口位置 i=1: 'idb'"
        W2["窗口: i d b\ncnt: {a:0,b:1,i:1,d:1,e:0}\ntarget: {a:1,b:1} → b多,a差"]
        style W2 fill:#ffcccc
    end

    subgraph "窗口位置 i=2: 'dba' ✅匹配!"
        W3["窗口: d b a\ncnt: {a:1,b:1,d:1,i:0,e:0}\ntarget: {a:1,b:1} → 匹配!"]
        style W3 fill:#ccffcc
    end

    W1 -->|"右移1位,\n移出i移入b"| W2 -->|"右移1位,\n移出d移入a"| W3

窗口内字符计数对比表：

窗口字符 a b d e i 状态
目标 s1 1 1 0 0 0 —
“eid” 0 0 1 1 1 ❌
“idb” 0 1 1 0 1 ❌
“dba” 1 1 1 0 0 ✅

优化过程

方案 思路 时间 空间
排序比较 每个子串排序 O(S₂ × S₁ log S₁) O(S₁)
字符计数 + 滑动窗口 固定窗口大小滑动，维护字符频数差 O(S₂ + 字符集) O(1)（数组 26 个）

优化核心：

  • 用 int[26] 计数数组代替 HashMap（字符集固定为小写字母）
  • 右指针扩展 → 更新计数；左指针收缩 → 回退计数
  • 每轮检查窗口是否与目标频数完全一致

最优解

复杂度分析

维度 复杂度 说明
时间 O(S₂) right 指针一次遍历，matches O(26) = O(1)
空间 O(1) 仅两个长度为 26 的整型数组
关键优化 不比较子串排序，而是比较频数数组是否相等，避免 O(S₁ log S₁) 开销
适用场景 字符串排列/变位词问题（Anagram）的经典模板

题目二十：最小栈（LeetCode 155）

题目描述

设计一个支持 push、pop、top 和获取最小元素 getMin 的栈，且所有操作均为 O(1)。

分类标签

栈 设计 数学 辅助结构

初步思路（暴力）

维护一个普通栈 + 一个记录当前最小值的变量，每次 pop/top 时遍历栈找最小值。

问题： getMin() 是 O(N)，不满足题目要求。

画图演示（栈内元素变化）

辅助栈法：主栈 + 最小栈同步维护

    graph TD
    subgraph "操作序列"
        OP1["push(-2)"]
        OP2["push(0)"]
        OP3["push(-3)"]
        OP4["getMin() = -3"]
        OP5["pop() → 移除-3"]
        OP6["top() = 0"]
        OP7["getMin() = -2"]
    end

    subgraph "主栈 + 最小栈状态"
        S1["主栈: [-2]\n最小栈: [-2]"]
        S2["主栈: [-2,0]\n最小栈: [-2,-2]"]
        S3["主栈: [-2,0,-3]\n最小栈: [-2,-2,-3]"]
        S4["主栈: [-2,0]\n最小栈: [-2,-2]"]
    end

    OP1 --> S1 --> OP2 --> S2 --> OP3 --> S3
    S3 --> OP4 --> OP5 --> S4 --> OP6
    S4 --> OP7

详细元素状态：

操作 主栈 最小栈 说明
push(-2) [-2] [-2] 最小栈同步入 -2
push(0) [-2, 0] [-2, -2] 0 不小于当前最小 -2，最小栈重复入 -2
push(-3) [-2,0,-3] [-2,-2,-3] -3 < -2，最小栈入 -3
pop() [-2,0] [-2,-2] 同时 pop
top() 返回 0 —
getMin() 返回 -2 — 最小栈顶即当前最小

优化过程

方案 思路 push/pop getMin 额外空间
暴力遍历 每次 getMin 遍历找最小 O(1) O(N) O(1)
辅助栈 主栈 + 最小栈同步维护 O(1) O(1) O(N)
差值法 只存与最小值的差值 O(1) O(1) O(1)（需 long）

辅助栈（推荐）： 每个元素入主栈时，同步将当前最小值入辅助栈；出栈时同步 pop。空间换时间，getMin 直接读辅助栈顶。

最优解

复杂度分析

维度 复杂度 说明
时间 push O(1) 只涉及两个栈的 push
时间 pop O(1) 只涉及两个栈的 pop
时间 getMin O(1) 直接读 minStack 栈顶
空间 O(N) 最坏情况下 minStack 与 dataStack 等大
核心技巧 minStack 重复压入当前最小值，保持两栈高度同步，pop 时无需额外判断

本篇总结

题号 题目 核心数据结构 关键技巧 时间 空间
16 合并 K 个有序链表 优先队列（小根堆） K 路归并，堆顶始终最小 O(N log K) O(K)
17 数据流的中位数 双堆（大根堆 + 小根堆） 左堆存小的一半，右堆存大的一半 O(log N) O(N)
18 序列化与反序列化二叉树 BFS 队列 null 也入队，保证重建顺序 O(N) O(N)
19 字符串的排列 滑动窗口 + int[26] 固定窗口大小，频数数组比较 O(S₂) O(1)
20 最小栈 主栈 + 辅助最小栈 最小栈同步维护，getMin O(1) O(1) O(N)

核心套路一览：

  • 堆/优先队列：全局最值维护（Top K、合并有序序列）
  • 双堆：数据流中位数、分区平衡问题的万金油
  • 滑动窗口：固定/可变窗口解决子串/子数组问题
  • 辅助数据结构：栈+辅助栈、队列+visited 集合
  • BFS/DFS：树/图的序列化与遍历
