claude-hud：给 Claude Code 装上 HUD 仪表盘

发表于2026-03-23|更新于2026-03-23|开源项目
|浏览量:

项目概览

claude-hud 是由开发者 jarrodwatts 构建的 Claude Code 插件，通过在终端界面叠加一层实时 HUD（Head-Up Display，抬头显示器），让开发者对 AI 编程助手的内部运行状态一目了然。

定位 Claude Code 可视化监控面板

在长时间使用 Claude Code 编程时，你是否曾好奇：当前消耗了多少 context？正在调用哪些工具？有哪些 agent 在并行工作？todo 列表推进到哪一步了？ 原生界面并不直接展示这些信息，而 claude-hud 解决了这个痛点。

核心功能：HUD 各面板详解

claude-hud 将 Claude Code 的运行状态拆分为多个透明面板叠加在终端上，整体效果类似赛车游戏的抬头显示器——关键信息尽收眼底，无需切换视图。

📊 Context 使用面板

实时显示当前 session 的 context 消耗情况：

  • 已用 Token 数：直观数值 + 百分比进度条
  • Context 上限：可配置阈值，超限时高亮预警
  • 消耗趋势：简单折线图展示近几次输出的 context 增长速率

痛点解决：过去只能靠”凭感觉”判断 context 是否即将耗尽，现在有数据支撑决策。

🔧 工具调用面板

展示当前正在或最近调用的工具列表：

状态标识：✓ 成功 / ⏳ 执行中 / ✗ 失败

🤖 Agent 状态面板

当 Claude Code 启动多个子 agent 并行工作时，此面板显示：

  • 活跃 agent 数量
  • 每个 agent 的当前任务描述
  • agent 间的层级关系（父 agent → 子 agent）

✅ Todo 进度面板

追踪 todo 工具的完成状态：

  • 总任务数 / 已完成数 / 进行中数
  • 每个 todo item 的描述和状态
  • 进度百分比环形图

实现原理

claude-hud 的实现涉及以下几个关键层面：

1. Claude Code Plugin 机制

Claude Code 提供了 plugin/ 目录规范，claude-hud 通过监听内部的 stream 事件来获取运行数据。这是官方支持的扩展点，无需 fork 或 patch Claude Code 本身。

2. 状态订阅与聚合

插件内部维护一个状态订阅中心，Claude Code 运行时的各类事件（工具调用、context 变化、agent 启动/终止）通过事件流汇入，HUD 组件订阅这些数据并触发局部渲染更新。

核心类 approximately：

3. 终端 UI 渲染

使用 Blessed 或类似库在终端上绘制”透明覆盖层”，在不干扰主输出的前提下叠加 HUD 面板。覆盖层采用 overlay 模式，渲染优先级高于普通 stdout 输出。

与 Claude Code 原生界面对比

维度 Claude Code 原生 claude-hud
Context 可见性 ❌ 无直接显示 ✅ 实时数值 + 进度条
工具调用追踪 纯文本日志，分散 ✅ 面板聚合，状态清晰
Agent 监控 ❌ 不可见 ✅ 层级树状展示
Todo 进度 需要主动查询 ✅ 常驻面板
学习成本 低 ✅ 无额外学习

结论：claude-hud 并不替代原生界面，而是作为”信息增强层”叠加上去，适合对 AI 行为有深度好奇心或需要精细调优 context 用量的开发者。

技术亮点

  1. 零侵入扩展：利用官方 Plugin API，不依赖内部未公开接口，更新 Claude Code 版本后插件大概率仍能正常工作。

  2. 高效渲染：终端 UI 库选用遵循最小刷新原则，仅在状态变更时重绘对应面板区域，避免频繁全屏刷新导致的闪烁。

  3. 可配置阈值：context 警告线、刷新频率、面板布局均可自定义，满足不同工作流偏好。

  4. 轻量级依赖：核心依赖仅 1-2 个（主要是终端 UI 渲染库），安装体积小，初始化速度快。

总结

claude-hud 解决了一个很实际的问题：AI 编程助手越来越强大，但它是一个”黑盒”，开发者往往不知道内部正在发生什么。 通过将 context、工具、agent、todo 等关键指标可视化，claude-hud 让 AI 编程从”盲目信任”走向”透明协作”。

如果你经常使用 Claude Code，推荐安装体验——11,584 个 stars 已经证明了它的实用价值。
