browser-use: 让 AI Agent 操控浏览器的开源利器

发表于2026-03-22|更新于2026-03-22|AI Agent
|浏览量:

当 AI Agent 能像人一样操作浏览器，自动化测试、网页数据采集、智能填表将彻底改写游戏规则。

项目概览

browser-use/browser-use 是一款将大语言模型（LLM）与浏览器自动化深度融合的开源框架，GitHub 斩获 36k+ stars。它的核心理念是：让 AI agent 通过自然语言指令直接操控浏览器，无需人类干预即可完成复杂的多步骤网页任务。

从数据抓取、自动化测试，到 AI 驱动的在线操作，browser-use 正在重新定义”浏览器自动化”的边界。

核心特性

1. 自然语言驱动的浏览器控制

传统的 Selenium/Playwright 需要工程师手写精确的 DOM 定位器（CSS Selector、XPath），browser-use 则将这一步交给 LLM。开发者只需描述目标：

Agent 会自动完成：打开浏览器 → 导航到 GitHub → 输入凭据 → 执行搜索 → 点击 star。LLM 理解页面结构，自主决策下一步操作。

2. 多 Provider 支持

支持 OpenAI、Anthropic、Azure OpenAI、Ollama 等多种 LLM Provider，通过统一的接口抽象层切换模型。这种灵活性让开发者可以在不同场景下选择成本/性能最优的方案。

3. 视觉理解 + 文本理解双轨并行

Agent 不仅读取 HTML 文本，还能”看见”页面截图，结合视觉大模型理解复杂 UI。这解决了传统爬虫无法处理的 CAPTCHA、图片验证码等难题。

4. 可扩展的 Action System

框架内置了丰富的原子操作（点击、输入、滚动、截图、上传文件等），同时支持自定义 Action，开发者可以注册自己的业务动作，实现高度定制化。

5. Python SDK + WebUI + Cloud 三位一体

  • Python SDK：核心库，pip install 即可使用，代码量仅需几行
  • WebUI：提供可视化任务编排界面，非技术人员也能配置任务
  • Cloud：托管服务，免部署即可运行大规模浏览器任务

适用场景

场景 说明
智能数据采集 替代传统爬虫，AI 理解页面结构，应对复杂反爬策略

RPA（机器人流程自动化） 财务对账、订单处理、表单填写等重复性网页操作

局限性

  1. LLM 成本与延迟：每一步操作都调用 LLM API，大型任务耗时长且费用不低
  2. 复杂交互瓶颈：对需要精确时序（如拖拽、多选日期控件）的页面支持仍有提升空间
  3. 反爬对抗：部分网站使用高级反爬机制（行为指纹、TLS 指纹），browser-use 无法绕过
  4. 稳定性：生产级长时间运行任务可能出现页面状态不一致，需要额外的错误重试机制
  5. 调试困难：Agent 自主决策过程不透明，出问题时排查链路较长
