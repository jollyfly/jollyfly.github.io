【每日一项目】Manifest - OpenClaw 智能 LLM 路由插件

发表于 2026-03-22 分类于 每日一项目

项目概览

Manifest 是一个 OpenClaw 插件，通过 23 维评分算法在 <2ms 内将用户请求路由到最合适的大模型，实测节省成本最高可达 70%。

很多场景下我们用 GPT-4o 或 Claude Opus 处理一个”今天天气怎么样”这类简单查询，Manifest 解决的就是这个资源浪费问题。

  • GitHub: mnfst/manifest
  • 开源协议: MIT
  • 语言: TypeScript
  • 支持模型: 300+（OpenAI、Anthropic、Google、DeepSeek、xAI、MiniMax 等）

核心特性

特性 说明
🔀 LLM 路由 23 维评分算法，<2ms 决策，调用最适合的模型
🔄 自动降级 模型调用失败时，自动切换到备选模型
🔔 用量告警 设置阈值，超出后主动通知
📊 实时仪表盘 令牌消耗、费用、消息量、模型使用一览
🔒 隐私优先 本地模式下数据完全不外传

架构：Cloud vs Local

Manifest 提供两种部署模式：

Cloud 版（默认）

  • 快速安装，多设备访问
  • 仅传输 OpenTelemetry 元数据（模型、token 数、延迟），消息内容永不收集

Local 版

  • 所有数据留在本机
  • 支持 Tailscale 远程访问

评分算法原理

这是 Manifest 最核心的部分——23 维评分算法。官方没有完全公开细节，但文档透露了评估维度的大致分类：

维度类别 示例
查询复杂度 任务类型、上下文长度
领域匹配度 代码/数学/创意/闲聊
实时性需求 是否需要最新信息
精确度要求 容错率 vs 准确性
成本权重 模型的 token 单价

每次请求进来，算法在 <2ms 内完成评分，决定该走哪个模型。整个过程发生在本地，无需调用第三方。

对比 OpenRouter

Manifest OpenRouter
架构 本地路由，数据不外传 云端代理，所有流量经过其服务器
费用 免费（插件本身） 每 API 调用收 5% 手续费
源码 MIT，全开放 闭源
隐私 盲代理，物理上无法读取你的 prompt 需信任第三方
模型覆盖 300+ 300+

支持的模型（部分）

  • OpenAI: gpt-5.3, gpt-4.1, o3, o4-mini 等 54+
  • Anthropic: claude-opus-4-6, claude-sonnet-4.5 等 14+
  • Google: gemini-2.5-pro, gemini-2.5-flash 等 19+
  • DeepSeek: deepseek-chat, deepseek-reasoner 等 11+
  • MiniMax: minimax-m2.5, minimax-m1 等 5+
  • 本地: Ollama（Llama、Gemma、Mistral 等）

我的看法

Manifest 解决的是一个很实际的问题：LLM 调用成本控制。在实际项目中，简单查询用贵模型是最大的浪费。

亮点：

  1. 和 OpenClaw 无缝集成，安装即用，不需要改代码
  2. 本地模式隐私保障强，这对企业用户很有吸引力
  3. 算法透明，开源后可以自行审计

需要关注的地方：

  1. 23 维评分算法没有完全开源（只在本地模式下可审计）
  2. Cloud 版需要注册 API Key，有一定的云服务依赖

总体来说，推荐在 OpenClaw 生产环境中部署，成本控制效果是可量化的。

下一步

官方提供了完整的 贡献指南，包括开发环境搭建和架构笔记。有兴趣深入研究的可以去看看评分类的具体实现。
