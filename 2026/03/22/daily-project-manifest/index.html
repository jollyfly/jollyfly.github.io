项目概览

Manifest 是一个 OpenClaw 插件，通过 23 维评分算法在 <2ms 内将用户请求路由到最合适的大模型，实测节省成本最高可达 70%。

很多场景下我们用 GPT-4o 或 Claude Opus 处理一个”今天天气怎么样”这类简单查询，Manifest 解决的就是这个资源浪费问题。

  • GitHub: mnfst/manifest
  • 开源协议: MIT
  • 语言: TypeScript / NestJS / SolidJS
  • 架构: Monorepo (Turborepo)
  • 支持模型: 300+（OpenAI、Anthropic、Google、DeepSeek、xAI、MiniMax 等）

核心特性

特性 说明
🔀 LLM 路由 23 维评分算法，<2ms 决策，调用最适合的模型
🔄 自动降级 模型调用失败时，自动切换到备选模型
🔔 用量告警 设置阈值，超出后主动通知
📊 实时仪表盘 令牌消耗、费用、消息量、模型使用一览
🔒 隐私优先 本地模式下数据完全不外传

技术架构分析

Monorepo 结构

Manifest 源码采用 Turborepo 管理，拆分为三个独立包：

包 技术栈 职责
frontend SolidJS 实时仪表盘（成本/令牌/模型使用）
backend NestJS 评分算法 API、OTLP 接收、订阅管理
openclaw-plugin TypeScript OpenClaw 钩子注册、路由拦截、遥测上报

这种拆分的好处是：插件端只负责拦截和上报，评分逻辑在 backend，仪表盘在 frontend，三者解耦独立迭代。

源码解析

1. 路由决策：routing.ts

这是 Manifest 最核心的文件，负责在每次请求中决定”该用哪个模型”。

Momentum 缓存机制

设计思路：同一个对话 session 内的连续请求，30分钟内复用之前的评分结果（只记录 tier 层级），避免重复调用远程评分接口。这个缓存按 sessionKey 隔离，不同用户互不影响。

消息过滤（关键优化）

为什么这样设计？

  • system 和 developer 消息通常很长（系统 prompt 可达几千 token），但它们只影响模型行为，不反映用户任务的真实复杂度
  • 只取最近10条消息，是因为对话早期内容和当前任务相关性已经很低
  • DTO 精简后只传 role + content，减少网络传输体积

resolveRouting 函数签名

返回值包含 reason 字段，这是给用户的可解释性输出，方便在仪表盘看到”为什么选了这个模型”。

2. 全链路可观测：hooks.ts

Manifest 使用 OpenTelemetry（OTel） 实现全链路追踪，将每一次 LLM 请求都拆成 span 记录。

架构亮点

1. 拦截式，而非修改式

Manifest 不修改 OpenClaw 的核心代码，而是通过钩子（hooks）拦截 message_received、before_agent_start、agent_end 等生命周期事件。这种方式：

  • 升级 OpenClaw 版本时几乎不会冲突
  • 插件可以热插拔，不影响主程序

2. Blind Proxy 隐私设计

Cloud 模式下，插件将请求转发给 Manifest 的盲代理（Blind Proxy）。这个代理只知道”要调哪个模型、传了多少 token”，但物理上无法解密 prompt 内容，因为加密发生在插件层，代理只有密文。

3. 23维评分算法的工程实现

从源码可见，算法实际上分为两部分：

  • 本地部分（插件端）：消息过滤 → 构建 DTO → 调用 /api/v1/routing/resolve
  • 远程部分（backend）：23维评分 → 返回 tier + model

评分算法本身托管在 Manifest 的云服务中，这是他们闭源的核心商业部分。

4. 错误自愈机制

路由失败时有降级策略（fallback），通过 momentum 缓存机制，如果 resolve 接口超时（3s），会使用上一次成功的 tier，不会阻塞请求。

适用场景与局限性

适用

  • 多模型并存的 OpenClaw 生产环境
  • Token 费用需要精细化管控的团队
  • 对数据隐私有要求但又想用云端评分服务的场景

局限

  • 评分算法核心闭源，无法在本地审计
  • Cloud 版有云服务依赖，离线环境只能用 local 模式（但 local 模式功能受限）
  • 目前只支持 OpenClaw 生态，通用性有限

对比 OpenRouter

Manifest OpenRouter
架构 本地路由 + 盲代理 云端代理，所有流量经过其服务器
费用 免费（插件本身） 每 API 调用收 5% 手续费
源码 MIT，全开放 闭源
隐私 盲代理，物理上无法读取 prompt 需信任第三方
可观测性 全链路 OTel + 实时仪表盘 基础用量统计
