核心框架：二叉树 DFS 遍历

逐题解析

21. LC104 二叉树的最大深度

思路：递归求左右子树最大深度 + 1。

复杂度：O(n) 时间，O(h) 递归栈空间（h 为树高）。

关键点：深度优先遍历的本质是后序遍历（左→右→根）。

22. LC112 路径总和

思路：从根到叶子节点的路径和是否等于 target。

关键点：递减 targetSum，到叶子节点判断是否为 0。

23. LC113 路径总和 II

思路：找所有从根到叶子 sum==target 的路径。

关键点：记录完整路径 + 回溯。路径总和 II 是 22 的升级版，需要枚举所有路径。

24. LC257 二叉树的所有路径

思路：DFS 记录路径，到叶子节点把路径加入结果。

关键点：字符串路径构造技巧，避免手动拼接 ->。

25. LC222 完全二叉树的节点数

思路：完全二叉树节点数 = 2^h - 1（满） + 最底层节点。

关键点：利用完全二叉树的性质（左深度==右深度→满二叉树），将 O(n) 优化到 O(log n × log n)。

通用解题框架

遇到二叉树问题，先判断：

  1. 需要前/中/后序哪种遍历？
  2. 需要返回什么（单个值 / 路径 / 布尔）？
  3. 是否需要记录路径（→ 回溯）？
