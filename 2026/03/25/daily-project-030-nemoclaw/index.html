daily project | NemoClaw 在 NVIDIA OpenShell 中安全运行 AI Agent

项目概览

NemoClaw（NVIDIA/NemoClaw，⭐16,414）是由 NVIDIA 官方开源的参考技术栈，旨在 NVIDIA OpenShell 运行时中更安全地运行 OpenClaw 始终在线 AI 助手。它将 OpenClaw Agent 部署在一个由 Landlock + seccomp + netns 三层内核安全机制构成的隔离沙箱中，所有网络请求、文件系统访问和推理调用都受到声明式安全策略的约束。

核心定位：用 NVIDIA 官方背书的容器化安全架构，解决 AI Agent 权限失控、推理请求裸奔这两个生产级痛点。

核心特性

1. 多层内核级沙箱隔离

NemoClaw 不依赖单一安全机制，而是叠加了三层纵深防御：

隔离层 技术实现 防护目标
Landlock Linux 内核 5.13+ 安全模块，基于 BPF 的文件系统/网络访问控制 限制 Agent 对文件系统树和网络栈的读写权限
seccomp 系统调用过滤白名单 阻止 ptrace、mount、syslog 等危险系统调用
netns 网络命名空间隔离 Agent 网络请求必须经过 OpenShell 网关，无法直接连接外部

这套组合拳使得即便 Agent 被恶意 Prompt 注入，也难以突破沙箱边界。

2. 声明式安全策略（Policy-as-Code）

安全规则以 YAML/声明式配置形式存在，由 OpenShell 网关在运行时强制执行：

策略变更 → 重新应用 → 网关热加载，无需重启 Agent 进程。

3. 托管推理（Managed Inference）

传统 AI Agent 直接持有 API Key，所有推理请求从用户的网络环境发出。NemoClaw 颠覆了这一架构：

关键意义：API Key 和凭证留在用户本地（~/.nemoclaw/credentials），推理请求由 OpenShell Gateway 代理转发，Agent 进程本身永远接触不到原始 Key。

4. 零信任推理路由

NemoClaw 在沙箱创建前会验证推理 Provider 的可达性：

  • OpenAI 兼容 Provider：先尝试 /responses API，失败则降级到 /chat/completions
  • Anthropic 兼容 Provider：尝试 /v1/messages
  • 验证失败则阻止沙箱创建，先修复再继续

这避免了 Agent 启动后才发现推理不可用的”死后调试”困境。

5. 一键安装与编排

安装脚本自动完成：

  • Node.js 20+ 环境检查与安装
  • Docker 容器运行时验证
  • OpenShell 运行时拉取
  • 引导向导（sandbox 命名、模型选择、策略配置）
  • 沙箱镜像推送（镜像约 2.4 GB 压缩）

Blueprint 的四阶段生命周期

Blueprint 是版本化的 Python artifact，负责沙箱创建的端到端编排：

  1. Resolve：从 NVIDIA GitHub Releases 或指定 URL 下载 Blueprint artifact
  2. Verify：校验 artifact SHA256 digest，防止供应链攻击
  3. Plan：调用 OpenShell CLI 规划所需资源（CPU/RAM/磁盘/网络策略）
  4. Apply：通过 OpenShell CLI 在 Docker/k3s 中实例化沙箱

资源约束与 OOM 防护

安装文档明确指出沙箱镜像推送时（Docker daemon + k3s + OpenShell Gateway + 解压缓冲区同时运行），内存压力可能导致 OOM。建议：

  • 最低 8 GB RAM，推荐 16 GB
  • 低于 8 GB 的机器配置等量 swap（以性能换稳定）

适用场景

最适合

  1. 企业内 AI 助手部署：对数据安全有合规要求（不允许 API Key 外流、不允许 Agent 直连外网），NemoClaw 的托管推理 + egress 策略是天然解法
  2. 高风险 Prompt 注入场景：客服机器人、文档问答等直接面向用户的 Agent，Landlock + seccomp 双重隔离防止逃逸
  3. 多租户 AI 平台：每个租户一个独立沙箱，网络和文件访问完全正交，资源隔离清晰
  4. DGX Spark 边缘节点：DGX Spark 有特殊的 cgroup v2 和 Docker 配置要求，NemoClaw 提供专门的 spark-install.md 覆盖这些细节

不适合

  • 个人开发者快速原型（安装依赖较重，8 GB RAM 最低要求）
  • macOS Intel 机型（暂不支持 Podman，只有 Apple Silicon 通过 Colima/Docker Desktop 支持）
  • Windows 非 WSL2 环境

总结

NemoClaw 是 NVIDIA 将 OpenClaw AI Agent 纳入 Agent Toolkit 生态 的关键棋子。它不只是一个安装脚本，而是一套从内核安全隔离（Landlock/seccomp/netns）到声明式策略管理再到托管推理的完整安全架构。对于需要在生产环境大规模部署 AI Agent 的团队，这是目前最接近”开箱即用”的 NVIDIA 官方方案。

当前为 Alpha 状态（2026-03-16 预览），接口和 API 可能在后续版本中变更，但核心安全模型已经稳定，值得在受控环境中积极测试。
