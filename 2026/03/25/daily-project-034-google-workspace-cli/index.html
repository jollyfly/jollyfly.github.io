项目概览

Google Workspace CLI（googleworkspace/cli，npm 包名 @googleworkspace/cli，命令名 gws）是 Google 官方开源的一个命令行工具，用一个 CLI 覆盖 Google Workspace 全产品线：Drive、Gmail、Calendar、Sheets、Docs、Chat，以及 85+ 其他 Google API 端点。

⭐ 22,424 GitHub Stars | 动态构建（无静态命令列表）| 40+ Agent Skills | Node.js + Rust | MIT License

核心设计哲学：gws 不维护任何 API 规范的静态列表。它在运行时从 Google Discovery Service 拉取规范，动态构建整个命令树。这意味着当 Google 发布新的 API 端点时，gws 无需任何更新就能直接使用。

适用场景

非常适合的场景

  1. AI Agent 的 Google Workspace 集成

    • Agent 需要读写 Google Docs、发送 Gmail、查询 Calendar
    • gws 的 JSON 输出 + SKILL.md = Agent 无需任何定制开发就能用

  2. 批量办公自动化

    • “每天自动生成销售报告并 email 给团队”
    • “扫描 Gmail 特定发件人，自动归档到 Drive 文件夹”
    • “批量创建 100 个 Calendar 会议邀请”

  3. CI/CD 中的 Google Workspace 操作

    • 部署完成后自动在 Google Chat 发通知
    • 定时导出 Sheets 数据到备份位置

  4. 运维 / DevOps 工程师的快速探索

    • gws schema xxx 快速查看任何 API 的完整 schema
    • 不需要翻 Google API 文档，直接 CLI 探索

需要注意的点

  • 仍处于活跃开发中：v1.0 之前有 breaking changes 的可能性
  • OAuth 测试模式限制：未验证的 OAuth App 只能申请约 25 个 scope，如需 85+ scope 需要 Google 验证或选择子集
  • Service Account 需要 Google Workspace 管理员权限才能代表域内用户

快速上手

总结

Google Workspace CLI 的核心价值在于消除 Google Workspace 集成的所有摩擦。传统的 REST API 调用需要：阅读文档 → 获取 OAuth → 写 curl 命令 → 处理分页 → 解析响应。gws 把这个流程压缩为：安装 → gws auth setup → gws <service> <resource> <method>。

而对于 AI Agent 来说，更重要的是：结构化 JSON 输出 + SKILL.md + 动态 schema 发现 构成了一个完美的 Agent 集成层。Agent 不需要知道 Gmail API 怎么认证、Sheets 怎么批量写入——它只需要调用 gws gmail messages send --to xxx --subject yyy 并解析 JSON 结果。

“For humans — stop writing curl calls against REST docs. For AI agents — every response is structured JSON.”

这是 Google Workspace CLI 区别于所有其他 Google API 工具的本质差异。
