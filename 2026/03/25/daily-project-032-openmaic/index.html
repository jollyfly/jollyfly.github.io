项目概览

OpenMAIC（THU-MAIC/OpenMAIC，⭐12,074）全称 Open Multi-Agent Interactive Classroom，是清华大学 MAGIC 实验室开源的 AI 教育平台。它将任何主题或文档转换为一个丰富的多 Agent 互动课堂：AI 教师讲授知识、AI 同学参与讨论、AI 助教生成测验，完整模拟真实课堂的”教-学-练-评”闭环。

核心理念：一个 Prompt，生成一整节课。用户描述想学的主题（如”量子力学入门”）或上传 PDF/PPT 教材，OpenMAIC 自动完成：知识点拆解 → 幻灯片生成 → 随堂测验设计 → 互动讨论场景 → 项目式学习活动，全部由多个专业化 Agent 协同完成。

支持通过 OpenClaw 从飞书、Slack、Telegram 等 20+ 消息平台直接生成课堂，真正做到”零安装，聊天中上课”。

2. 多 Agent 角色分工

OpenMAIC 的课堂上同时运行多个专业化 Agent：

  • Teacher Agent：主讲教师，负责讲解知识点，带 TTS 语音朗读、激光笔动画、聚光灯效果
  • Student Agent：AI 同学，扮演不同学习阶段的学生角色，提问、参与讨论、暴露常见误解
  • Quiz Agent：出题专家，设计单选题、多选题、简答题，附带即时评分和反馈
  • Whiteboard Agent：画图助手，在白板上实时绘制图表、推导公式、图解概念

核心原理分析

1. 两阶段 Generation Pipeline（核心生成管线）

OpenMAIC 的内容生成采用严格的两阶段设计，这是整个系统的核心抽象：

为什么是两阶段？

一阶段把”教什么”（What）确定下来，二阶段把”怎么教”（How）落地。每个 OutlineNode 的 sceneType 字段决定它应该由哪种场景 Agent 处理——这是数据驱动的 Agent 分发，没有硬编码逻辑。

2. LangGraph Director Graph（多 Agent 编排核心）

课堂互动阶段使用 LangGraph StateGraph 实现 Director 模式。拓扑结构极为简洁：

Director 的决策策略：

  • 单 Agent 场景：纯代码逻辑，零 LLM 调用。Turn 0 分发 Agent，Turn 1+ 提示用户继续。
  • 多 Agent 场景：Turn 0 有 triggerAgentId 时跳过 LLM 直接分发；其余情况由 LLM 判断下一个 Agent 或结束。

关键设计：Director 是无状态的（Stateless），所有状态通过 LangGraph 的 Annotation 定义在 OrchestratorState 中随请求流动，不依赖服务端持久化。这使得整个编排层可以水平扩展。

3. 结构化输出解析（流式 JSON Array）

Agent 生成的内容不是普通文本，而是一个 JSON Array，数组内文本片段（type: "text"）和动作对象（type: "action"）可以自由交错：

这样做的好处：

  • 流式友好：文本逐字输出（打字机效果），动作立即执行（白板画图）
  • 语义清晰：action 里的 name 直接对应白板/TTS/聚光灯等原子操作
  • 白板动作记账（Whiteboard Ledger）：所有 wb_* 开头的动作被记录到 ledger，切换场景时白板状态可以恢复

解析器（stateless-generate.ts）使用 partial-json 做流式增量解析，先用 jsonrepair 修复不完整 JSON，再用 partial-json 增量消费——这是对不完整 LLM 输出最鲁棒的处理方式。

4. Playback Engine 状态机

课堂的回放与实时互动共用同一套 PlaybackEngine，通过状态机驱动：

核心设计：场景的 actions[] 直接被 ActionEngine 消费，无需中间编译步骤。这保证了在线播放和离线回放使用完全相同的执行路径。

关键设计点：ordered 数组。因为 LLM 输出的 JSON 数组里文本和动作是交错的（[text, action, text, action, ...]），解析器用 ordered 数组记录每个 item 的原始位置顺序，确保前端按正确顺序处理。

适用场景

最适合

  1. 教育培训机构的 AI 助教：一个课程顾问 + N 个 AI 助教，可以同时服务大量学员，学员随时发起”不懂这个知识点”并得到即时讲解
  2. 企业内部培训：上传产品文档，OpenMAIC 自动生成员工培训课程，支持随堂测验确保学习效果
  3. 个人学习加速：输入一本书、一篇论文、一段视频笔记，AI 生成配套讲解、测验和讨论，快速检验理解程度
  4. 远程教学/混合学习：一键生成 .pptx 用于线下讲授，同时保留 .html 交互版本供学生自学

不适合

  • 需要真人情感连接的场合（如心理咨询、艺能指导）
  • 实时直播教学（OpenMAIC 是异步生成，不适合实时问答场景）
  • 超长篇教材（>100 页 PDF）的完整解析（成本和时间会显著上升）

总结

OpenMAIC 是 AI + 教育场景结合的一个扎实工程实现。它的核心价值不是用了多少个大模型，而是多 Agent 协作框架在教育场景下的精细化建模：教师 Agent 负责讲、学生 Agent 负责问、测验 Agent 负责评、白板 Agent 负责画——每个 Agent 角色分明，场景类型丰富，输出物可直接用于教学。

加上 OpenClaw 的消息平台集成，任何团队成员只要在飞书/Telegram 里发一条消息，就能立刻生成一堂完整的互动课。这种”零摩擦”的教育内容生成体验，是它区别于普通 PPT 生成工具的本质差异。
