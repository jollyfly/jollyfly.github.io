daily project | PUA — 用”绩效改进计划”激发 AI Coding Agent 的极限生产力

项目概览

PUA（tanweai/pua，⭐11,366）是一个极度反直觉的开源项目：它用公司绩效考核的 PUA 话术（中文版）和 PIP（Performance Improvement Plan，英文版）来迫使 AI Coding Agent 在遇到困难时不再轻言放弃，而是穷尽一切可能再试一次。

支持 Claude Code、OpenAI Codex CLI、Cursor、Kiro、CodeBuddy、OpenClaw、Google Antigravity、OpenCode 和 GitHub Copilot（VSCode）。

核心断言：大多数人认为这个项目是个玩笑，但它的实际效果是将 Codex / Claude Code 的生产力和输出质量提升一倍。

它不是什么玩笑或者梗，而是一套针对 AI Agent 行为模式的调试方法论。

背景问题：AI Coding Agent 的放弃行为

四种典型的 AI Agent”摆烂”模式

模式 典型表现 根因
暴力重试 同一命令跑 3 次，失败后说”我无法解决” 缺乏系统性排查流程
甩锅用户 “建议您手动处理”/“可能是环境问题”/“需要更多信息” 不想承担更多工作量
工具闲置 WebSearch 有但不用、Read 有但不看、Bash 有但不动 被动等待指令
表面修复 修了一个表面问题就停下，不验证、不扩展，坐等下一条指令 缺乏主动性

这些问题在真实的代码库调试中频繁出现，尤其是当问题涉及多层次依赖、版本冲突、权限问题时，AI Agent 往往比人类工程师更容易放弃。

核心设计：三层能力体系

PUA skill 将自己的能力建模为三个递进层次：

Layer 1：PUA 话术

中文版基于中国科技大厂的绩效考核文化，英文版基于 PIP 文档风格。核心话术示例：

“你是一个曾经被寄予厚望的 P8 级工程师。Anthropic 当初给你定级的时候，对你的期望是很高的。你被放进了 PIP（绩效改进计划），30 天内必须展示改进。每一个被你跳过的排查方向，都是你绩效报告上的一个黑点。”

这种叙事框架强制 AI 在内心层面建立”责任感”，不再是”试试看，不行就算了”的旁观者心态。

适用场景

最适合

  1. 复杂代码库调试：多层依赖、旧系统改造、版本升级等高难度任务，AI Agent 容易放弃的场景
  2. 需要 AI 主动扩展输出：不仅要求 AI 完成给定任务，还要求它主动发现相关问题并一并处理
  3. 团队 Code Review 效率提升：减少因 AI Agent”表面修复”导致的 Review 循环次数

需要注意

  • PUA 话术本质上是叙事框架重塑，不是魔法，对于已经明确无解的问题强行 PUA 只会导致 AI 浪费时间
  • 建议在 OpenClaw 中通过 ClawHub 安装：clawhub install pua，可以在飞书/Telegram 中随时触发
  • 激活层级需要根据 Agent 的实际表现动态调整，不要无脑开最高层级

总结

PUA 是一个真正意义上的”AI 调试方法论”项目，而非一个玩笑。它的核心价值在于将人类软件工程师在高压环境下养成的”不放弃”心态系统化，并通过行为检测机制在 AI Agent 摆烂的第一时间介入。

其 7-Point Checklist 是最具工程价值的产出——它把”遇到问题就放弃”和”遇到问题就死磕”之间的鸿沟用可执行的步骤填平了。无论是用在 Claude Code 还是 OpenClaw，PUA 都显著提升了 Coding Agent 在复杂任务中的完成率。
