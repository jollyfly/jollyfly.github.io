项目概览

AutoResearch（karpathy/autoresearch）是深度学习泰斗 Andrej Karpathy 开源的一个极简 autonomous AI research 实验框架。核心思想简单而有力：给 AI Agent 一个真实可运行的单 GPU LLM 训练环境，让它在夜里自主调参、训练、评估，早上你来收实验报告。

⭐ 54,584 GitHub Stars | 单文件 agent 修改 | 5 分钟固定时间预算 | 纯 PyTorch 实现

Karpathy 本人在 2026 年 3 月的访谈中坦言自己”大概从 12 月开始就没手动写过一行代码了”，AutoResearch 正是他展示这一范式的技术载体——把科研人员从调参炼丹的体力活中彻底解放出来。

核心特性

1. 极简双文件架构

整个 repo 只包含三个真正重要的文件，设计上刻意保持最小化：

文件 职责 谁修改
prepare.py 数据下载、BPE Tokenizer 训练、DataLoader、评估工具 固定不变
train.py GPT 模型定义 + Muon+AdamW 优化器 + 训练循环 Agent 唯一修改对象
program.md Agent 指令 prompt（元认知层） Human 迭代优化

这种设计的精妙之处在于：Agent 的作用域被严格限制在 train.py 一个文件内。这意味着：

  • Diff 可审计、可回滚
  • Agent 不可能意外破坏数据管道或评估逻辑
  • 实验对比永远公平（同数据、同 tokenizer、同评估协议）

2. 5 分钟固定时间预算

无论你在 H100 还是 MacBook M3，训练严格运行 5 分钟 wall-clock time（不含启动和编译开销）。这产生了两个关键优势：

3. val_bpb — 架构无关的评估指标

Validation Bits Per Byte（val_bpb） 是核心评估指标：

  • 来自信息论的严格度量，与词汇表大小无关
  • 可公平比较任何架构变化（增加 attention head、换激活函数、改位置编码）
  • 越低越好，代表模型对数据的压缩效率更高

技术架构分析

整体架构图

GPT 模型架构（train.py 核心）

关键设计选择：

  1. WINDOW_PATTERN — 支持 "L"（全 causal attention）和 "SSSL"（Sparse Sliding Window Local + Sparse Global）两种注意力模式。SSSL 模式通过交替的 banded attention pattern 大幅降低 attention 计算复杂度：

  2. DEPTH 参数 — 控制模型深度，所有其他参数（FFN 维度、参数量）都是 DEPTH 的函数。Agent 只需调这一个参数就能控制模型复杂度。

Muon 优化器实现

Muon 的核心优势在于它直接优化矩阵流形（matrix manifold），而非在欧几里得空间做梯度下降。实验表明，在 ImageNet 和 LLM 训练任务中，Muon 相比 AdamW 有显著收敛速度优势。

适用场景

非常适合的场景

  1. 科研机构快速探索架构变体

    • 想验证某个 attention 变体是否有效？丢给 AutoResearch，让 Agent 跑一晚上，明天拿结果。
    • 多 GPU 集群太贵？单 H100 也能持续产出有价值的实验数据。

  2. 个人开发者学习 LLM 训练

    • 极简代码 + 固定时间预算 = 完美的学习环境
    • 改一个参数，看效果，理解训练 dynamics

  3. 嵌入式/端侧 LLM 调优

    • 在 Jetson、MacBook、M2 Mac Mini 上跑
    • Agent 自动为你的硬件找到最优模型配置

  4. AI Agent 研究本身

    • 把 AutoResearch 本身作为研究对象
    • 迭代 program.md，研究”什么样的元认知指令能让 Agent 更高效”

不适合的场景

  • 需要分布式训练的大模型（>1B 参数）
  • 需要长时间（>5 分钟）才能看到效果的优化策略
  • 对实验结果可复现性有严格要求的场景

快速上手

总结

AutoResearch 的本质是一个极简研究自动化框架——它用最小化的代码基础设施（一个固定文件 + 一个可变文件 + 一个 prompt），构建了一个完整的 autonomous research 闭环。Karpathy 的设计哲学贯穿其中：约束产生创造力。Agent 只改 train.py 一件事，反而让它比开放式探索更高效。

“The agents claim that we are now in the 10,205th generation of the code base…” — @karpathy

这个 repo 的存在本身，就是对 AI research 未来的一次严肃探索。
