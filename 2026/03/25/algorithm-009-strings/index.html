字符串解题框架

LC3 无重复字符的最长子串

暴力解 O(n²)

两个 for 循环枚举所有子串，再用 Set 判断是否有重复：

复杂度：外层 O(n)，内层 O(n)，总体 O(n²)，每个子串还要 O(n) 检查。

优化解 O(n) — 滑动窗口 + 哈希

用左右指针维护一个无重复的窗口，右指针扩展，遇到重复字符时左指针收缩。

graph LR
    A["s = 'abcabcbb'"] --> B["a b c a b c b b"]
    B --> C1["L=0, R=0, set={}"]
    C1 --> C2["R=1, add 'a', set={a}"]
    C2 --> C3["R=2, add 'b', set={a,b}"]
    C3 --> C4["R=3, add 'c', set={a,b,c}"]
    C4 --> C5["R=4, 遇到'a'重复 → L右移到1
set={b,c,a}, max=3"]
    C5 --> C6["继续扩展...max最终=3"]

优化思路：用 lastIndex 记录字符上次出现位置，遇重复直接跳到 lastIndex + 1：

复杂度：双指针各遍历一次，O(n) 时间，O(1) 空间（128 固定数组）。

LC5 最长回文子串

暴力解 O(n³)

枚举所有子串中心，判断是否为回文：

复杂度：枚举 O(n²)，每步判断 O(n)，O(n³)。

优化解 O(n²) — 中心扩散

回文串从中心向外扩散有奇数长度和偶数长度两种：

graph LR
    A["中心扩散法"] --> B["奇数中心：'aba'
中心=b, 扩散向左右"]
    A --> C["偶数中心：'aa'
中心=两字符间, 扩散向左右"]
    
    B --> D["找到最长回文"]
    C --> D

复杂度：外层 O(n)，每次扩散最坏 O(n)，O(n²) 时间，O(1) 空间。

最优解 O(n) — Manacher 算法（了解）

用特殊符号 # 预处理，将奇偶统一处理，O(n) 可求解最长回文，但实现较复杂，面试频率低。

LC20 有效的括号

栈解法 O(n)

左括号入栈，右括号匹配栈顶：

graph TD
    A["s = '({[]})'"] --> B["'(' 入栈"]
    B --> C["'{' 入栈"]
    C --> D["'[' 入栈"]
    D --> E["']' 匹配栈顶'[' ✓ pop"]
    E --> F["'}' 匹配栈顶'{' ✓ pop"]
    F --> G["')' 匹配栈顶'(' ✓ pop"]
    G --> H["栈空 → true"]

复杂度：遍历一次，O(n) 时间，O(n) 空间（最坏全左括号）。

LC49 字母异位词分组

排序 + 哈希 O(nk log k)

将每个字符串排序作为 key，相同 key 的为一组：

graph LR
    A["eat", "tea", "tan", "ate", "nat", "bat"]
    A --> B["排序后：
ate→aet
eat→aet
tea→aet
tan→ant
nat→ant
bat→abt"]
    B --> C["哈希分组"]
    C --> D["{aet:[ate,eat,tea]}
{ant:[tan,nat]}
{abt:[bat]}"]

最优解 O(nk) — 计数哈希

中心扩散 O(n²)

和 LC5 类似，对每个位置做奇数/偶数中心扩散：

最优解 O(n) — Manacher 或 动态规划

用 DP 思想：dp[i][j] = (s[i]==s[j]) && (j-i<2 || dp[i+1][j-1])

本章总结

题型 关键技巧 最优复杂度
无重复最长子串 滑动窗口 + 记录上次出现位置 O(n)
最长回文子串 中心扩散（奇/偶） O(n²)
有效括号 栈匹配 O(n)
字母异位词分组 排序或计数作 key O(nk)
回文子串计数 中心扩散或 DP O(n²)

核心思想：字符串问题很多是数组问题的延伸，滑动窗口、栈、哈希、双指针同样适用。回文问题特有的中心扩散是重点。
