项目概览

gstack（garrytan/gstack）是 Y Combinator 现任 CEO Garry Tan 开源的个人 AI 软件工厂。他用 gstack 在 60 天内产出了 60 万行生产级代码（35% 测试覆盖），其中一周就写了 14 万行 net LOC，兼职状态、同时全职运营 YC。

⭐ 45,441 GitHub Stars | 15 个 Slash Commands | 8 个原子化工具 | MIT License

这不是一套提示词模板，而是一个完整的 AI 软件工程组织系统——gstack 把 Claude Code 变成了一支虚拟工程团队：CEO、工程师经理、设计师、QA 负责人、安全官、发布工程师，全部通过 /slash commands 驱动。

核心特性

1. 角色扮演系统 — 从空白提示词到结构化职责

传统的 AI coding assistant 给你一个空白的 prompt 框。gstack 给你 15 个预设角色，每个角色都有明确的职责边界和协作协议：

2. 两种安装模式：Repo-Local vs User-Global

关键设计：真实文件 commit 到你的 repo（不是 git submodule），所以 git clone 即插即用，没有任何额外依赖。

3. 安全护栏技能（Hook-Based Safety）

这些是基于 hook 的安全兜底，在非 Claude 平台上也能以纯文本 advisory prose 形式内联显示，不会因为平台差异而失效。

4. 多 Agent 兼容

gstack 基于 SKILL.md 标准，理论上任何支持该标准的 Agent 都能使用：

  • ✅ Claude Code（原生支持）
  • ✅ Codex（通过 --host codex 安装）
  • ✅ OpenClaw
  • ✅ Cursor
  • ✅ 其他 SKILL.md 兼容 Agent

  1. 生成各平台的 Agent 特定技能文件（Codex 格式 vs Claude Code 格式）
  2. 注册 SKILL.md 到 Agent 的技能发现系统
  3. 设置本地 hook 脚本

源码实现深度解析

gstack 与 OpenClaw 的生态关系

Garry Tan 在 README 中特别提到：OpenClaw 的作者 Peter Steinberger 用 AI agents 独立构建了 247K GitHub Stars 的项目。gstack 深受 OpenClaw 的 SKILL.md 架构启发，并在设计上有意兼容。

关键实现细节

1. Skill 自动发现机制

gstack 技能的安装路径遵循 Agent 的标准约定：

Agent 启动时扫描 ~/.claude/skills/*/SKILL.md 并自动注册。无需重启，技能实时可用。

2. /browse 替代 MCP Chrome 工具

gstack 的 /browse skill 是对 Anthropic MCP mcp__claude-in-chrome__* 工具的直接替代。Garry Tan 的 CLAUDE.md 配置中明确写明：

这是一个刻意的主权宣示：用 gstack 自己的浏览器控制能力替代平台提供的 MCP 工具，避免平台更新导致的破坏性变更。

3. 多 Agent 的 skill 格式差异处理

适用场景

非常适合的场景

  1. 技术创始人 / solo founder

    • Garry Tan 就是代表：全职 CEO + 兼职写代码，两不耽误
    • gstack 让”有想法就能快速验证”成为现实

  2. 小团队（3-10人）引入 AI 辅助工程流程

    • 把 /review / /qa / /ship 标准化到 PR 流水线
    • 让 AI 扮演”永不疲倦的高级工程师”

  3. AI coding assistant 新手

    • 空白 prompt 无从下手 → 直接用 /office-hours 开始
    • gstack 的 slash commands 就是最佳实践的内化

  4. 企业安全合规场景

    • /guard + /freeze 提供操作级别的安全护栏
    • 不依赖平台特性，hook-based 安全在任意 Agent 上都可迁移

需要注意的点

  • 需要 Claude Code / Codex：本质上是一个 Claude Code 插件生态，不是通用的 AI Agent 框架
  • Mac/Linux 优先：Windows 需要额外配置（Node.js + Git for Windows + bash）
  • CLAUDE.md 配置复杂度：需要正确配置 browse 技能路由才能避免冲突

快速上手

总结

gstack 的本质是一个工程化最佳实践的系统化封装。Garry Tan 20 年的工程经验 + 他在 YC 观察数千家初创公司的视野，全部内化为这套 slash commands 系统。

“2026 — 1,237 contributions. 2013 — 772 contributions. Same person. Different era. The difference is the tooling.”

这句话值得所有工程师认真思考：工具塑造了你能解决的问题的规模。gstack 展示的不是”AI 能帮你写代码”，而是”AI 能帮你运营一个完整的软件工程组织”。
