项目概览

Paperclip（paperclipai/paperclip）是”如果 OpenClaw 是一个员工，那 Paperclip 就是一家公司”——它是一个用 Node.js + React 构建的 AI Agent 编排平台，核心目标是让一群 AI Agent 像真实公司一样运作：有组织架构、有预算管控、有目标对齐、有审计日志。

⭐ 32,708 GitHub Stars | 开源自托管 | Node.js + React | PostgreSQL

与市面上的 Agent 框架（LangChain、CrewAI 等）不同，Paperclip 不告诉你怎么构建 Agent，它只管理 Agent 所在的”组织”。它的参照物不是”更强大的 ChatGPT”，而是”一家能 24/7 自主运行的 AI 公司”。

核心特性

1. 组织架构（Org Chart）驱动

Paperclip 的核心抽象是公司，而非”一组并行的 Agent”。每个公司有：

每个 Agent 有明确的上级（Supervisor）、同级协作协议、预算上限。任务通过组织层级向上汇报、向下指派。

2. 心跳式调度（Heartbeat Scheduling）

持久化状态是 Paperclip 区别于”开着 20 个 Claude Code Tab”的关键：Agent 的工作上下文跨心跳保持，不需要每次重新加载。

3. 原子化执行与预算强制

不会有 double-work，不会有预算超支。 这两个问题是在家运行多个 Agent 的人最常踩的坑，Paperclip 用数据库事务从根本上解决。

4. 多公司隔离（Multi-Company Isolation）

每个实体的所有数据都是 company-scoped 的，这是面向”AI 公司投资人”或”AI 孵化器”的场景——用一套部署管理多个 AI 公司组合，每个公司的数据完全隔离。

5. 全链路审计日志（Immutable Audit Log）

每个操作都有不可篡改的日志，包含：时间戳、Agent ID、Tool Calls、输入输出、Token 消耗、Supervisor 审批记录。

适用场景

非常适合的场景

  1. AI 初创公司 / solopreneur 运行多个 AI Agent 业务线

    • “我有 3 个 AI 公司在跑，分别做 SaaS、内容创作、电子商务”
    • Paperclip 就是你的 AI 集团管控平台

  2. 需要严格成本管控的 Agent 部署

    • 每个 Agent 有月度预算上限（可设 $5~$500）
    • 防止 Agent 进入无限循环消耗大量 token

  3. 需要审计合规的业务

    • 金融、医疗、法律等行业的 AI 决策需要完整日志
    • Paperclip 的 immutable audit log 天然满足合规要求

  4. 需要 24/7 自主运营的业务

    • 客服、社媒运营、数据监控
    • 人在睡觉，AI 公司在跑

不适合的场景

  • 只有 1-2 个 Agent：Paperclip 的复杂度需要多 Agent 协作才能体现；单个 OpenClaw 实例可能更合适
  • 需要拖拽式工作流：Paperclip 没有 DAG/pipeline builder，它模型的是公司而非流水线
  • 需要实时响应用户输入：Paperclip 是调度系统，不是对话系统

快速上手

先决条件：Node.js 20+ , pnpm 9.15+

总结

Paperclip 的本质是一个自治组织的操作系统。它的核心洞察是：

“Not a chatbot. Not an agent framework. Not a workflow builder. Not a prompt manager. A company made of agents.“

这四个”Not”精准划清了 Paperclip 与所有竞品的边界。它不构建 Agent（那是 OpenClaw、Claude Code 的活），它构建的是 Agent 存在的社会结构——组织、目标、预算、审计。21 世纪的公司形态，正在从”人类员工 + 金字塔组织”向”AI Agent 群 + 目标导向网络”演化，Paperclip 是这个演化路径上最早的开源基础设施之一。
