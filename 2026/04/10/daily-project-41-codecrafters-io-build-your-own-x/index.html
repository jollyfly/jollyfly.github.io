项目描述： Master programming by recreating your favorite technologies from scratch.

核心特性

  1. “造轮子”学习法：通过亲手实现 Redis、Docker、Kubernetes、Git 等流行工具来深度理解它们
  2. 难度分级挑战：每个 Challenge 都有多个 Stage，从基础到高级逐步加难度
  3. 实战替代空学：不是看教程，而是真正写代码、通过测试才能解锁下一关
  4. 多语言支持：多数 Challenge 支持用多种语言提交实现
  5. 自动评测系统：提交后自动运行测试用例，即时反馈

挑战的 Stage 设计模式

以 “Build Your Own Git” 为例，典型的 Stage 递进：

每个 Stage 的交付物：

  1. 规格说明（SPEC.md）：明确要实现什么
  2. 测试用例：Codecademy 平台提供自动化测试
  3. 参考实现：提供 Go/Python 等语言的参考方案

为什么”从零实现”是最高效的学习方法？

传统的学习方式（看教程、抄代码）存在两个根本问题：

  1. 被动接收：大脑只需要识别模式，不需要生成模式
  2. 表层理解：知道”是什么”，不知道”为什么”

从零实现强制解决这两个问题：

  • 必须生成代码：没有任何参照代码可抄，只能从原理出发推导
  • 遇到边界情况：测试用例覆盖了各种边界，会暴露自己没考虑到的场景
  • 深度理解原理：实现完才能真正理解为什么 Redis 用 RESP 协议、Githubit 的对象模型是这样的

Git 实现的精髓

通过这个 Challenge，你将亲手实现 Git 的核心概念：

理解了这个模型后：

  • Git 的分支不过是 refs/heads/xxx 文件里存的一个指针
  • git checkout branch 就是把 HEAD 指针移动到新的 commit
  • git merge 就是把两个分支的 commit 链合并

Codecademy 平台的商业模式

Codecademy 的 CLI 工具（codecrafters）是开源的，任何人都可以用它搭建自己的评测平台：

平台会自动：

  1. 在隔离容器中运行你的代码
  2. 执行测试用例
  3. 验证输出是否符合规格
  4. 公布结果和排行榜
