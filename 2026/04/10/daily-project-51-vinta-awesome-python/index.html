源码深度剖析

一、README 解析器——用 markdown-it 构建 AST

文件：website/readme_parser.py

这是整个网站的数据源头。README.md 是 Markdown 格式，程序需要把它转成结构化数据（分类 > 子分类 > 条目）。核心用的是 markdown-it-py 构建语法树（AST）。

关键设计决策：

  1. 使用 AST 而非正则：正则处理 Markdown 容易出错（尤其是嵌套结构），AST 能精确识别标题、链接、代码块。markdown-it 的 SyntaxTreeNode 将 Markdown 解析为树形结构，每个节点有 type、content、children。

  2. 双渲染模式：render_inline_html（网站展示）和 render_inline_text（数据处理）共用同一 AST，一次解析两处使用。

  3. escape 保护：markupsafe.escape 防止 XSS 注入——用户写 <script> 会被转义成 &lt;script&gt;，安全。
