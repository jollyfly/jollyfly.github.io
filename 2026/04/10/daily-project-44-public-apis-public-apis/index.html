设计决策分析：

  1. HEAD 请求优先：比 GET 更轻量，只取响应头不下载体，CI 运行速度快 10 倍以上
  2. 超时保护：timeout=5 防止被慢服务器卡住整个 CI 流水线
  3. 重定向跟随：allow_redirects=True 检测最终 URL，防止 HTTP 跳转
  4. HTTPS 强制：最终 URL 必须是 HTTPS，从源头拒绝 HTTP 不安全 API

为什么 CORS 标注很重要？

对于前端开发者来说，如果 API 不支持 CORS（cors: “no”），那么在浏览器中直接调用会失败。很多初学者不知道这个问题的原因，CORS 标注让开发者能在选型阶段就排除不合适的 API。
