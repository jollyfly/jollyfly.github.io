项目描述： AI驱动的渗透测试助手，运行在 Parrot OS 上，完全使用本地 LLM（Ollama + metatron-qwen），无需云端 API Key，无需订阅，100% 离线可用。

核心特性

🤖 本地 AI 分析引擎

核心模型 metatron-qwen 是基于 huihui_ai/qwen3.5-abliterated:9b 微调的专有渗透测试模型，通过 Ollama 本地运行，支持 16,384 token 上下文窗口。强调无幻觉原则：AI 被要求仅基于实际扫描输出做出判断，不得猜测版本或推断未验证的 CVE。

🔍 自动化侦察管线

内置集成六种主流侦察工具：

  • nmap：服务版本检测 + 默认脚本（-sV -sC -T4 --open）
  • whois：域名注册信息、Registrar、IP 归属
  • whatweb：Web 技术指纹识别（CMS/框架/Web 服务器）， aggression level 3
  • curl：HTTP 头信息采集（自动尝试 HTTP 和 HTTPS）
  • dig：DNS 记录查询
  • nikto：Web 服务器漏洞扫描

🔁 Agentic 循环

AI 在分析过程中可主动调用额外工具或执行 DuckDuckGo 搜索，最多 9 轮工具循环。每个工具结果通过 LLM 二次压缩（摘要至 15 条安全要点），避免上下文溢出。

🌐 免 API Key 的 Web 搜索

通过 duckduckgo-search（DDGS 库）实现完全免费的 CVE 搜索和漏洞情报查询，同时直接抓取 MITRE CVE 数据库。

🗄️ MariaDB 全量记录

五张关联表完整记录扫描会话：history、vulnerabilities、fixes、exploits_attempted、summary，支持增删改查。

📤 报告导出

扫描结果可导出为 PDF（专业漏洞报告）和 HTML（浏览器可读格式）。

适用场景与局限性

适用场景

  • CTF / HackTheBox / VulnHub 本地练习环境渗透
  • 授权的企业内网评估（完全离线，不暴露目标数据）
  • 教育/培训 演示 AI + 安全工具的协作模式
  • Parrot OS / Kali 日常快速侦察

局限性

  • ⚠️ 不执行实际攻击：仅侦察 + AI 分析，无 Metasploit 集成
  • ⚠️ 模型依赖：需要 8GB+ RAM，4b 变体效果降级
  • ⚠️ agentic loop 上限 9 轮：复杂目标可能提前终止
  • ⚠️ 数据库凭证硬编码（password="123"）：生产环境严重风险
  • ⚠️ 无权限提升/横向移动能力

我的看法

METATRON 展示了 AI Agent + 安全工具 组合的工程可行性，而非依赖云端闭源 API。核心亮点是：

  1. ReAct Loop + 输出压缩 在有限上下文内实现了多轮渗透分析
  2. metatron-qwen 微调模型 针对渗透测试领域的输出格式做了专项优化
  3. 全本地化 设计在隐私敏感场景有独特价值

但整体完成度更接近 PoC/演示项目：数据库明文密码、无 Metasploit 集成、agentic loop 无动态终止条件、PDF 导出依赖 reportlab 而非专业报告模板。把它当作学习 AI + 安全结合的范本比直接用于生产渗透测试更合适。
