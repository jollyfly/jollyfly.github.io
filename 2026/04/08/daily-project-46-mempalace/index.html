MemPalace：96.6% 召回率的 AI 记忆系统，永久记住每一次对话

项目描述： The highest-scoring AI memory system ever benchmarked. And it’s free.

MemPalace 是一个本地运行的 AI 记忆系统，核心思想来自古希腊演说家西塞罗的”记忆宫殿”——将信息组织成可遍历的空间结构，而非依赖 AI 自行决定记住什么。它的 benchmark 成绩达到了 LongMemEval R@5 96.6%（零 API 调用），每年仅需约 $10 的 token 成本即可记住用户数月乃至数年的所有对话记录。

核心特性

  1. 记忆宫殿结构：用 Wing（翼）/Room（房间）/Closet（储物间）/Drawer（抽屉）的层级结构替代向量数据库的纯语义搜索，让 AI 知道”去哪里找”
  2. AAAK 压缩方言：专为 AI 设计的结构化符号语言，压缩比 30x，170 tokens 即可加载数月上下文
  3. 时序知识图谱：SQLite 存储实体关系链，带时间有效性（valid_from → valid_to），追踪”某个事实在何时为真”
  4. MCP 工具集：19 个 MCP 工具供 Claude/ChatGPT/Cursor 直接调用，无需手动操作 CLI
  5. 全本地运行：ChromaDB + SQLite，无需任何外部 API，离线可用
  6. Benchmarks SOTA：LongMemEval R@5 96.6%（免费方案最高），ConvMemBench、LoCoMoBench、MEMBench 均验证有效

核心模块解析

palace_graph.py — 记忆宫殿的图结构

这是整个系统最核心的模块。它没有使用任何外部图数据库，而是直接利用 ChromaDB 的 metadata 构建图结构：

关键思想：房间（Room）是图的节点，同一个 Room 横跨多个 Wing 时，它们之间就形成了 Tunnel（隧道）。Hall 则是同 Wing 内房间之间的走廊（连接关系）。

这个设计的精妙之处在于：它不依赖 LLM 来决定哪些信息重要。结构（哪个 Wing、哪个 Room）本身就带来了 34% 的检索提升。

searcher.py — 精确匹配而非语义幻觉

搜索返回的是 Drawer 内容（verbatim 原词），不是摘要。这是关键设计决策：压缩在 Closet（储物间）中完成，但 Drawer 永远保存原文，AI 永远不会因摘要而丢失细节。

dialect.py — AAAK 压缩方言

mcp_server.py — 19个 MCP 工具

安装方式：claude mcp add mempalace -- python -m mempalace.mcp_server

源码解析：ChromaDB 如何存储 Palace 结构

ChromaDB 的 metadata 字段（wing/room/hall/source_file/date）直接映射到 Palace 的层级结构，不需要额外的图数据库。图遍历（找 Tunnel、Hall）在 palace_graph.py 中通过对 metadata 的二次计算完成。

适用场景与局限性

适用场景：

  • 长期 AI 辅助开发（记住项目历史决策、技术选型理由）
  • 多角色协作（AI 记住不同人的背景、偏好、工作风格）
  • 个人知识管理（论文阅读笔记、项目复盘）
  • 需要严格数据隐私的用户（完全本地，不上云）

局限性：

  • 依赖 ChromaDB，向量检索本身有精度上限
  • AAAK 方言需要 AI 显式学习（虽然工具会自动教）
  • 初始化需要手动运行 mempalace init 和 mempalace mine
  • 房间自动检测（room_detector_local.py）质量依赖于 LLM 调用

我的看法

MemPalace 的核心洞察非常准确：AI 记忆的问题不是存储，而是检索的组织结构。大多数记忆系统让 AI 自己决定什么是重要的，这导致关键上下文被丢弃。

“记忆宫殿”的物理隐喻在这里不是噱头——它是真正的架构选择：结构化搜索 > 纯语义搜索。当 AI 知道”去哪个 Wing 找”的时候，准确率自然提升。

96.6% 的 benchmark 成绩、$10/年的成本、全本地运行的隐私保证，对于需要长期 AI 记忆的个人开发者或小团队来说，这是一个值得认真考虑的工具。
